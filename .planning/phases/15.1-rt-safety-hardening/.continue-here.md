## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Audio-path locks/heap/I/O | DO NOT introduce mutex acquisition, `new`/`delete`/`malloc`, file I/O, or any unbounded-latency call on any code reachable from JUCE `processBlock` / `audioDeviceIOCallback` / `getNextAudioBlock` / `NJClient::AudioProc`. The whole phase exists to remove these. | blocking | Use `realtime-audio-reviewer` agent on any audio-path code change. 15.1-10 audit grep enforces. |
| Audio-path `delete DecodeState*` | DO NOT call `delete` on `DecodeState*` from any audio-thread-reachable function. Use `deferDecodeStateDelete` (15.1-05 helper) — capture old pointer first, advance slot, then defer-delete. | blocking | Static helper is the only sanctioned free path; reviewer agent + 15.1-10 grep gate. |
| Raw back-pointers in `spsc_payloads.h` | DO NOT introduce `lc_ptr` / `user_ptr` / `Local_Channel*` / `RemoteUser*` raw fields in any SPSC payload. Codex HIGH-2; the file is FINAL since Wave 0. | blocking | M-9 grep gate fails build on regression; `spsc_payloads.h` has ONE commit ever (8f47965 from 15.1-04). |
| Audio-thread spin/sleep on full SPSC | DO NOT spin or sleep on the audio thread waiting for a full SPSC ring to drain. Use `try_push`, increment overflow counter, continue. | blocking | API only exposes `try_push`; counters exposed via getters; 15.1-10 asserts == 0. |
| Mutating mirror-key field outside publish path | DO NOT mutate `Local_Channel.channel_idx` outside `SetLocalChannelInfo`/`SetLocalChannelMonitoring`'s explicit add path. The audio-thread mirror is indexed BY this field; silent mutation strands the mirror entry. (Bug A and its duplicates.) | blocking | Architecture-mirror audit at 15.1-MIRROR-AUDIT.md; dead-but-live site at njclient.cpp:1648 flagged for v1.3 cleanup. |
| Direction-B stale legacy reader | DO NOT leave a legacy read site of a canonical field after migrating its writes to a mirror. UI snapshot publishers + direct accessors must be migrated TOO, or audio-correct features look broken (e.g. flat VU). | blocking | When migrating writes to mirror, grep for ALL legacy reads of the canonical field across `src/`, `juce/`, snapshot/accessor paths. |
| Direction-C decision-relocation drift | DO NOT translate the FORM of a legacy state-dependent decision (e.g. "alternate slots") when the new architecture forbids the decision's INPUTS. The faithful translation has no semantic content. Move the decision to a thread that has the inputs. | blocking | The 07a slot-picking-on-publisher mistake produced 3 separate bugs (every-other-interval, random bursty cutout). ### 15.1-09 (codec call-site integration, HIGH-1 / H-04 / CR-08 / CR-11-partial)

The architecturally hardest plan in the phase. `inversionAttachSessionmodeReader(ds)` runs on the run thread BEFORE every `PeerNextDsUpdate` publish; sets `ds->decode_fp = nullptr` and `ds->decode_buf = new DecodeMediaBuffer()`; pushes a `SessionmodeFileReader{file, buffer}` onto `m_sessionmode_file_readers`. `refillSessionmodeBuffers` runs every 20ms tick, reads up to 4 × 4 KB chunks per active FILE* into the corresponding DecodeMediaBuffer (lock-free SPSC push). The audio thread's `runDecode` reaches `decode_buf->Read` (lock-free SPSC drain) for ALL file-backed playback. **Invariant established:** every audio-thread-visible `DecodeState` has `decode_fp == nullptr` in shipped state. The H-04 `fread` at njclient.cpp:478 is structurally unreachable from the audio thread.

Two new drop counters added: `GetArmRequestDropCount()`, `GetSessionmodeRefillDropCount()` — both must be 0 for Signal 6.

### 15.1-08 (prealloc-hardening, M-01 / M-02 / M-03 / CR-11-partial)

`NJClient::SetMaxAudioBlockSize(int)` public API: throws `std::runtime_error` if `maxSamplesPerBlock > jamwide::MAX_BLOCK_SAMPLES` (Codex M-7); calls `tmpblock.Prealloc(maxSamplesPerBlock * sizeof(float))`. `JamWideJuceProcessor::prepareToPlay` wires the call in try/catch + latches `prevPreparedSize`. `processBlock` asserts `numSamples <= prevPreparedSize` in Debug builds (M-03; `((void)0)` in Release). `VorbisDecoder` ctor and `FlacDecoder::initDecoder` now `Prealloc` their WDL queues at construction (run thread per HIGH-1). CR-11 partial closure: WDL_TypedQueue layer CLOSED; libogg/libFLAC algorithmic residual deferred to 15.1-10 Instruments measurement (CONTEXT D-04).

### 15.1-07a post-UAT stabilization (3 fixes)

Real-DAW UAT on builds 284→288→289→290 caught and fixed three 07a regressions, all rooted in one architectural mistake:

1. **`b9899a0` / build 288** — Direction-B stale UI reader: `GetRemoteUsersSnapshot:4164-4165` and `GetUserChannelPeak:4388-4390` still read legacy `RemoteUser_Channel.decode_peak_vol` after audio-thread writes were migrated to `mirror.peak_vol_l/r`. Routed UI reads through `findRemoteUserSlot → mirror`.
2. **`e827453` / build 289** — every-other-interval audio cutout: 07a's `run_thread_next_ds_idx` XOR shadow alternated 0/1, but audio thread always reads `next_ds[0]` and shifts, so slot-1 publishes were unreachable until next shift. Mitigation: hardcode `useidx_to_publish=0` at all 3 publish sites.
3. **`e151dd8` / build 290** — random bursty cutout: build-289's always-slot-0 mitigation defer-deleted prior `next_ds[0]` on every publish, dropping data when run thread published 2+ updates between drains. **Final architectural fix:** moved slot picking from publisher to apply visitor (audio thread, has access to mirror state). Apply visitor logic = "fill empty slot 0; queue slot 1 if 0 occupied; defer-delete oldest only if both full" — recreates legacy `useidx = !!next_ds[0]` semantics on the correct thread.

The dead `run_thread_next_ds_idx` field at `njclient.cpp:544` is now unused; can be removed in v1.3 cleanup.
