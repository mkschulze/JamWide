## Decisions Made

- **Stored prelisten host/port in NinjamRunThread members:** The plan referenced `client->GetHostPort()` which does not exist in the NJClient API. Instead, the PrelistenCommand handler stores host/port from the command into `prelistenHost_`/`prelistenPort_` members, used when emitting PrelistenStatus::Connected in handleStatusChange. This is more reliable since it preserves the original command values.
- **Metronome restore on all exit paths:** Added metronome volume restoration not just in StopPrelistenCommand and ConnectCommand teardown (as planned), but also in the failure and async disconnect handlers in handleStatusChange. This prevents metronome from staying muted if a prelisten connection fails or is dropped.
