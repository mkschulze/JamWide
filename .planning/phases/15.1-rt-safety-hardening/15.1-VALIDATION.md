# Phase 15.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Source of truth: `15.1-RESEARCH.md` § "Validation Architecture" (D-07 / D-08 contract).

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Frameworks** | C++ unit tests via ctest (`build-test/` for fast no-JUCE loop, `build-tsan/` for TSan + JUCE) + manual UAT harness via `JamWideJuce_Standalone` |
| **Config files** | `build-test/CMakeCache.txt` (no TSan, no JUCE — fast unit loop); `build-tsan/CMakeCache.txt` (NEW — `--tsan` adds in 15.1-04: `JAMWIDE_BUILD_TESTS=ON + JAMWIDE_BUILD_JUCE=ON + -fsanitize=thread`) |
| **Quick run command** | `cd build-test && ctest --output-on-failure` |
| **Full suite command** | `./scripts/build.sh --tests && cd build-test && ctest --output-on-failure -j$(sysctl -n hw.ncpu)` |
| **TSan suite command** | `./scripts/build.sh --tsan && cd build-tsan && ctest --output-on-failure` (runs both NJClient unit tests AND JUCE-side OSC/MIDI tests under TSan — single target covers both audit scopes) |
| **TSan-standalone UAT** | After the build, launch `build-tsan/JamWideJuce_artefacts/Standalone/JamWide.app` (manual UAT, ≥ 5 minutes per sub-plan, ≥ 30 minutes for 15.1-10, against populated NINJAM server) |
| **Auditor re-run** | Spawn `realtime-audio-reviewer` agent over `src/core/njclient.cpp` and `juce/JamWideJuceProcessor.cpp` — required after every sub-plan |
| **Estimated runtime** | ctest ≈ 5–15s; full ctest under TSan ≈ 30–90s; standalone UAT ≥ 5 min per sub-plan, ≥ 30 min for 15.1-10 |

---

## Sampling Rate

- **After every task commit:** Run `cd build-test && ctest --output-on-failure` (quick).
- **After every plan wave:** Run full suite under TSan + targeted unit tests for that sub-plan. Spawn auditor agent.
- **Before `/gsd-verify-work`:** Full ctest + auditor re-run + manual TSan-standalone UAT must all be green.
- **Max feedback latency:** ctest < 90s; auditor agent ≈ 2–5 min; manual UAT ≥ 5 min per sub-plan, 30 min for phase close.
