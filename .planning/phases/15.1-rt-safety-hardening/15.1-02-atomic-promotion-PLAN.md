<objective>
Promote `m_beatinfo_updated`, `m_bpm`, `m_bpi`, and `m_interval_pos` from mutex-protected `int` to `std::atomic<int>`, eliminating the audio-thread acquisition of `m_misc_cs` at `src/core/njclient.cpp:825`. Closes AUDIT CR-03 and the bonus `m_interval_pos` race noted at AUDIT line 421.

**Codex L-10 response (incorporated in this revision):** the publication protocol for `m_beatinfo_updated` is **edge-triggered best-effort**, NOT a last-value latch. The writer always overwrites m_bpm/m_bpi with the LATEST values before raising the flag with release; the reader, on acquire-observing the flag, reads the LATEST payload (because the writer's last store wins) and clears the flag with relaxed. If the writer publishes N times faster than the reader runs, the reader sees only the LATEST values and misses the intermediate states. This is correct behavior — BPM/BPI are config values; only the most recent matters. Documented in code comment and verified by `test_missed_update_semantics`.
