# Phase 4 -- UI Design Contract

> Visual and interaction contract for Core UI Panels. Generated by gsd-ui-researcher, verified by gsd-ui-checker, iterated with visual mockups in Sketch.
>
> **Platform:** JUCE 8.0.12 (C++ desktop plugin / standalone). This is NOT a web project.
> All visual values are consumed by custom `LookAndFeel_V4` subclass and `paint()` overrides.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | JUCE LookAndFeel_V4 (custom subclass: `JamWideLookAndFeel`) |
| Preset | not applicable (native C++ rendering) |
| Component library | juce_gui_basics (Component, TextEditor, ListBox, ComboBox, Viewport) |
| Icon library | Custom SVG via `juce::Drawable::createFromSVG()` + BinaryData |
| Font | System sans-serif (juce::Font default). No external font dependency. |

**Design reference:** VB-Audio Voicemeeter Banana -- dark blue-gray pro-audio mixer. Dense single-screen layout, vertical channel strips with big faders, illuminated segmented LED VU meters.

**Primary focal point:** Big vertical faders with large circular grab handles showing dB values. The faders are the primary interaction element -- volume control must be fast and precise for live jamming.

---

## Window Dimensions

| Property | Value |
|----------|-------|
| Base size | 1000 x 700 pixels |
| Max size | ~1490 x 700 pixels (13 channel strips + master, chat hidden) |
| Resizable | Yes, **horizontal only** -- width dynamic based on channel count |
| Height | Fixed at 700px |
| Scale factors | 1.0x, 1.5x, 2.0x via `juce::AffineTransform::scale(factor)` |
| Scale UI access | Right-click context menu on connection bar |
| Minimum display | 1280x720 at 1.0x scale |

**Dynamic width rules:**
- Base: 1000px (min) -- fits ~6 collapsed strips + master + chat sidebar
- Grows automatically as groups expand or users join
- Max: ~1490px -- 13 channel strips + master, chat hidden
- Beyond 13 channels: horizontal scrollbar appears, window stays at max width
- Chat toggle adds/removes 260px

## Channel Strip Design (100px wide, VB-Audio Voicemeeter style)

### Strip Zones (top to bottom, filling full 626px height)

| Zone | Height | Contents |
|------|--------|----------|
| Header | ~66px | Name, channel info/codec, routing selector, subscribe/transmit |
| Fader + VU | **~516px** | Centered VU meter + big vertical fader with dB scale |
| Footer | ~38px | Pan slider + Mute/Solo buttons |

### Fader Design

| Property | Value |
|----------|-------|
| Style | VB-Audio Voicemeeter Banana -- big vertical fader |
| Fader track | 10px wide vertical groove, centered in strip |
| Fader thumb | 44px circular grab handle with green border |
| dB readout | Displayed ON the fader thumb, centered |
| Fill | Green (#2A7A4A) below thumb position |
| Travel | ~516px -- full strip height minus header/footer |
| Range | -∞ to +6 dB (0.0 to 2.0 linear) |
| dB scale ticks | +6, 0, -6, -18, -∞ -- subtle marks right of fader track |

### VU Meter Design

| Property | Value |
|----------|-------|
| Style | Segmented LED bars (L + R channels) |
| Width | 24px total (9px per channel + 6px gap) |
| Position | Centered in strip, left of fader track |
| Segments | 7px tall, 2px gap, fills available height |
| Colors | 0-70% green (#33CC33), 70-90% yellow (#E6B333), 90-100% red (#E63333) |
| Unlit | #0F1520 |
| Update rate | 30Hz timer |
| Ballistics | Attack 0.8, Release 0.92 (~300ms half-life) |

### Layout Within Strip

VU meter (24px) + gap (6px) + fader track (10px) = 40px total, **centered** in 100px strip width. Offset from left = (100 - 40) / 2 = 30px.

## Remote User Group Channels

Remote users with multiple channels use an **expand/collapse group pattern** (not tabs):

| State | Visual | Behavior |
|-------|--------|----------|
| Collapsed (▶) | Single strip showing first channel + "2ch" badge | User name, first channel controls visible |
| Expanded (▼) | Group strip + child strips slide out to the right | All channels visible simultaneously with own fader/VU/routing |

- **Single-channel users:** No expand arrow, no badge
- **Multi-channel users:** ▶/▼ arrow in header, `Nch` badge
- **Child strips:** Slightly lighter background (`surface-child` #252845), subtle green border when parent expanded
- **Expand/collapse:** Animates width change, other strips shift right/left

## Beat/Interval Progress Bar

|----------|-------|
| Height | 22px |
| Width | Full window width (stretches dynamically) |
| Segments | One per beat (BPI determines count) |

| Future beats | Dark (#0A0D18) |

## Interaction Contracts

### Connection Flow

| State | Connection Bar | Channel Area | Chat Panel |
|-------|---------------|--------------|------------|
| Disconnected | Fields editable, "Connect" | Dimmed placeholders, "Connect to a server to start jamming" | "Connect to a server to start chatting" |
| Connecting | Fields disabled, "Cancel", yellow dot | Same as disconnected | Same |
| License pending | Fields disabled, overlay | License dialog visible | Behind overlay |
| Connected | Fields disabled, "Disconnect", green dot, BPM/BPI | Strips populated with VU/faders | Chat active |
| Error | Fields re-enabled, "Connect", red dot | Disconnected state | Error message |

### Server Browser Flow

1. Click "Browse" → overlay + `RequestServerListCommand`
2. "Loading servers..." while waiting
3. List populates or error message
4. Single-click: fill server address. Double-click: fill + connect
5. Close / Escape: dismiss

### Chat Flow

1. Type + Enter or click "Send"
2. Parse via `parse_chat_input()` (ported from `src/ui/ui_chat.cpp`)
3. `SendChatCommand` pushed, input cleared
4. Incoming messages drained from `chat_queue` at 20Hz
5. Auto-scroll unless user scrolled up (jump-to-bottom pill)

### VU Meter Behavior

| Property | Value |
|----------|-------|
| Update rate | 30Hz timer |
| Attack | 0.8 (fast rise) |
| Release | 0.92 (~300ms half-life) |
| Segments | Green/yellow/red per threshold |
| Orientation | Vertical, bottom-to-top fill |

## Event/Command Communication

| Direction | Mechanism | Items |
|-----------|-----------|-------|
| UI → Run thread | `SpscRing<UiCommand, 256> cmd_queue` | Connect, Disconnect, SendChat, SetEncoderFormat, SetLocalChannelInfo, SetLocalChannelMonitoring, SetUserState, SetUserChannelState, RequestServerList |
| Run thread → UI | `SpscRing<UiEvent, 256> evt_queue` | StatusChanged, UserInfoChanged, TopicChanged, ServerList |
| Run thread → UI (chat) | `SpscRing<ChatMessage, 128> chat_queue` | ChatMessage structs |
| Run thread → UI (license) | `atomic<bool> license_pending` | License text |
| UI polling | Timer at 20Hz | Drains evt_queue + chat_queue, polls atomics |
