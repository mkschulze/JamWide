# Plan 14.1-01: Audio Prelisten Backend

**Objective:** Add command/event types, processor state, and NinjamRunThread prelisten handling so the main NJClient can connect in receive-only mode with zero local channels, auto-accept license, and auto-subscribe to all remote channels.

**Spec:** `docs/superpowers/specs/2026-04-13-audio-prelisten-design.md`

## Interfaces

- `PrelistenCommand { std::string host; int port = 2049; }` — queued to cmd_queue by UI
- `StopPrelistenCommand {}` — queued to cmd_queue by UI
- `PrelistenStateEvent { bool connected; std::string server_name; }` — pushed to evt_queue by run thread
- `processor.prelisten_mode` — `std::atomic<bool>`, checked by run thread + audio thread
- `processor.prelisten_volume` — `std::atomic<float>`, read by processBlock
