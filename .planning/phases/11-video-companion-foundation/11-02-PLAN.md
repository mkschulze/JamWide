**Step 2: Create docs/video/src/ws-client.ts** (per D-11, D-15).

WebSocket client that connects to `ws://127.0.0.1:{port}` (CRITICAL: use 127.0.0.1, NOT localhost -- mixed content pitfall from Research).

```typescript
import type { PluginMessage, ConfigMessage, RosterMessage } from './types';

export type WsCallbacks = {
  onConfig: (msg: ConfigMessage) => void;
  onRoster: (msg: RosterMessage) => void;
  onStatusChange: (connected: boolean) => void;
};

export function connectToPlugin(port: number, callbacks: WsCallbacks): WebSocket {
  // CRITICAL: Use 127.0.0.1, NOT localhost (mixed content blocking -- Research pitfall 1)
  const ws = new WebSocket(`ws://127.0.0.1:${port}`);

  ws.onopen = () => {
    callbacks.onStatusChange(true);
  };

  ws.onmessage = (event: MessageEvent) => {
    try {
      const msg: PluginMessage = JSON.parse(event.data as string);
      if (msg.type === 'config') {
        callbacks.onConfig(msg);
      } else if (msg.type === 'roster') {
        callbacks.onRoster(msg);
      }
    } catch {
      // Ignore malformed messages
    }
  };

  ws.onclose = () => {
    callbacks.onStatusChange(false);
  };

  ws.onerror = () => {
    callbacks.onStatusChange(false);
  };

  return ws;
}
```

No auto-reconnect per D-15. The reconnect button in the UI calls `connectToPlugin` again manually.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Companion page -> Plugin WS | Page connects to ws://127.0.0.1:7170 for config/roster |
| Companion page -> VDO.Ninja | Page loads VDO.Ninja in iframe for WebRTC video |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-06 | Spoofing | WebSocket client URL | mitigate | Hardcode ws://127.0.0.1 in ws-client.ts; never use user-supplied host. wsPort comes from URL query param but connects only to loopback. |
| T-11-07 | Information Disclosure | VDO.Ninja iframe | accept | VDO.Ninja is a third-party WebRTC service. IP disclosure is inherent to WebRTC and disclosed in privacy modal (Plan 03). Users acknowledge before reaching this page. |
| T-11-08 | Tampering | JSON message parsing | mitigate | Wrap JSON.parse in try/catch in ws-client.ts; malformed messages are silently ignored. No dynamic code execution or innerHTML with unsanitized data. Use textContent for DOM text updates. |
| T-11-09 | Information Disclosure | Room ID in URL | accept | Room ID is a SHA-1 hash, not reversible to server credentials. Visible in browser address bar but not sensitive. |
</threat_model>
