# Phase 12.1 Plan 01: Fix Video-Audio Sync Pipeline + Manual Delay Slider Summary

Dual-state buffer delay model replacing single cachedBufferDelay, with postMessage security fix (https://vdo.ninja origin), manual delay slider (0-30000ms), and 18 new unit tests covering state transitions.

## What Was Done

### Task 1: Dual-state delay model, pipeline fix, postMessage security, diagnostics, and tests (TDD)

**Commits:** 8fb8327 (RED), 86b8696 (GREEN)

Replaced the single `cachedBufferDelay` variable with a dual-state model:
- `lastAutoDelayMs`: Always updated when plugin sends bufferDelay (even in manual mode)
- `activeDelayMs`: The value currently applied to the iframe (auto or manual)
- `manualMode`: Boolean tracking current mode

Key behavioral changes:
- In auto mode, `setLastAutoDelay()` updates both `lastAutoDelayMs` and `activeDelayMs`
- In manual mode, `setLastAutoDelay()` only updates `lastAutoDelayMs` (silent tracking)
- `switchToAuto()` restores `lastAutoDelayMs` to `activeDelayMs` immediately
- No hardcoded 8000ms default anywhere; both start null
- `sendBufferDelayToIframe()` uses `'https://vdo.ninja'` targetOrigin instead of `'*'`
- `VideoSync:` prefixed diagnostic logging at every pipeline hop
- `buildVdoNinjaUrl()` uses `activeDelayMs` (omits `&buffer=` when null)
- `reapplyActiveDelay()` replaces `reapplyCachedBufferDelay()` (sends activeDelayMs on iframe reload)

Created `video-sync.test.ts` with 18 tests covering:
- lastAutoDelayMs tracking and overwriting
- Initial null state (no hardcoded default)
- Auto mode: setLastAutoDelay updates both values
- Manual mode: setLastAutoDelay only updates lastAutoDelayMs
- Mode switching (manual sets value, auto restores, auto-with-null)
- buildVdoNinjaUrl buffer param inclusion/omission
- getDelayDisplayText formatting (null, 0ms with "No delay", auto/manual labels)
- postMessage targetOrigin security
- reapplyActiveDelay behavior

Updated existing `buffer-delay.test.ts` with new function names and origin assertion.

## Decisions Made

1. **Dual-state model over single cache**: Two separate delay values (lastAutoDelayMs for plugin truth, activeDelayMs for applied value) prevent mode-switch bugs where auto/manual transitions lose state.
2. **Null default instead of 8000ms**: Plugin is sole source of truth. Pre-config state shows "--" in footer with disabled slider, preventing user interaction before delay is known.
3. **Specific postMessage origin**: Using `'https://vdo.ninja'` instead of `'*'` mitigates cross-origin information disclosure (T-12.1-02).
4. **Legacy export aliases**: `applyBufferDelay`, `getCachedBufferDelay`, `resetCachedBufferDelay`, `reapplyCachedBufferDelay` aliased to new names for backward compatibility with any code referencing old API.
