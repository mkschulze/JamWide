# Phase 4 -- UI Design Contract

> Visual and interaction contract for Core UI Panels. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform:** JUCE 8.0.12 (C++ desktop plugin / standalone). This is NOT a web project.
> All visual values are consumed by custom `LookAndFeel_V4` subclass and `paint()` overrides.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | JUCE LookAndFeel_V4 (custom subclass: `JamWideLookAndFeel`) |
| Preset | not applicable (native C++ rendering) |
| Component library | juce_gui_basics (Component, TextEditor, ListBox, ComboBox, Viewport) |
| Icon library | Custom SVG via `juce::Drawable::createFromSVG()` + BinaryData |
| Font | System sans-serif (juce::Font default). No external font dependency. |

**Design reference:** VB-Audio Voicemeeter Banana -- dark blue-gray pro-audio mixer. Dense single-screen layout, vertical channel strips, illuminated VU meters.

---

## Window Dimensions

| Property | Value |
|----------|-------|
| Base size | 1000 x 700 pixels |
| Resizable | No (`setResizable(false, false)`) |
| Scale factors | 1.0x (1000x700), 1.5x (1500x1050), 2.0x (2000x1400) |
| Scale method | `juce::AffineTransform::scale(factor)` on content component |
| Scale UI access | Right-click context menu on title/connection bar area |
| Minimum display | 1280x720 at 1.0x scale (comfortable margin) |

**Source:** D-22, D-23 (CONTEXT.md). Base size is Claude's discretion per RESEARCH.md recommendation.

## Layout Structure

All panels visible simultaneously. No tabs, no navigation. (D-01)

```
+------------------------------------------------------------------+
|  CONNECTION BAR (full width x 48px)                               |
|  [Server___________] [User_____] [Lock] [Connect] | Status | Codec|
+----------------------------------------------+-------------------+
|  CHANNEL STRIP AREA (left, flexible)         |  CHAT PANEL       |
|                                              |  (right, 280px)   |
|  +------+ +------+ +------+ +------+        |                   |
|  | LOCAL| | USR1 | | USR2 | | USR3 |        |  [message list]   |
|  | strip| | strip| | strip| | strip|        |                   |
|  |      | |      | |      | |      |        |                   |
|  | [VU] | | [VU] | | [VU] | | [VU] |        |                   |
|  | label| | label| | label| | label|        |                   |
|  +------+ +------+ +------+ +------+        |  [input] [Send]   |
+----------------------------------------------+-------------------+
```

| Region | Position | Dimensions (at 1x) |
|--------|----------|---------------------|
| Connection bar | top, full width | 1000 x 48 px |
| Channel strip area | below bar, left | 720 x 652 px |
| Chat panel | below bar, right | 280 x 652 px |
| Server browser overlay | centered modal, semi-transparent scrim | 600 x 500 px |
| License dialog overlay | centered modal, semi-transparent scrim | 500 x 400 px |

**Source:** D-01 through D-06 (CONTEXT.md), RESEARCH.md window dimensions recommendation.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | VU meter bar gap, icon-to-text inline gap |
| sm | 8px | Field padding, channel strip internal margins |
| md | 12px | Connection bar element spacing, chat message vertical gap |
| lg | 16px | Channel strip horizontal gap, panel internal padding |
| xl | 24px | Panel-to-panel gap (channel area to chat boundary) |
| 2xl | 32px | Section header spacing (topic bar, status area) |

Exceptions:
- Connection bar height: 48px (fixed, contains all connection controls)
- Channel strip width: 100px minimum (adjusts based on available space and user count)
- VU meter width: 24px (dual-bar, 10px per channel + 4px gap)
- Touch/click target minimum: 28px height for all interactive elements (buttons, fields, dropdowns)

## Typography

All sizes reference JUCE `juce::Font` constructed with `juce::FontOptions(size)`. System sans-serif.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px | Regular (plain) | 1.4 | Chat messages, server browser entries, labels |
| Label | 11px | Regular (plain) | 1.2 | Channel strip names, status indicators, timestamps |
| Heading | 16px | Bold | 1.2 | Panel titles ("Chat"), overlay headings ("Server Browser", "License Agreement") |
| Status | 14px | Bold | 1.0 | BPM/BPI display, connection state text, codec indicator |

**Rationale:** Dense pro-audio layout requires compact text. 13px body matches Voicemeeter Banana's information density. Bold reserved for status-critical information and section headings only.

| Interval progress | Custom painted | Thin horizontal bar below connection bar, shows beat/interval position. 2px height |

### Chat Panel (`ChatPanel`)

| Element | Type | Details |
|---------|------|---------|
| Topic display | `juce::Label` | Server topic at top of panel. `text-secondary` color. Wraps if long |
| Message viewport | `juce::Viewport` + custom `Component` | Custom-rendered message list. Color-coded per type. Word-wrapping. Auto-scroll (D-16, D-17) |
| Jump-to-bottom indicator | Custom painted | Arrow-down pill at bottom of viewport. Appears when scrolled up. Click scrolls to newest (D-17) |
| Chat input | `juce::TextEditor` | Single-line. Placeholder: `"Type a message..."`. Enter key sends |
| Send button | `juce::TextButton` | Label: "Send". Width: 50px |

**Chat formatting:** `HH:MM <sender> message` for regular, `HH:MM *** content` for system/join/part/topic, `HH:MM [PM from sender] message` for private, `HH:MM * sender action` for /me. Port `format_line()` from `src/ui/ui_chat.cpp`.

### Channel Strip (`ChannelStrip`)

Phase 4 delivers label + VU meter only. Faders, knobs, mute/solo buttons are Phase 5.

| Element | Type | Details |
|---------|------|---------|
| User/channel label | `juce::Label` | Top of strip. Truncate with ellipsis if name exceeds strip width. `text-primary` color |
| Codec badge | Custom painted | Small tag below label: "[FLAC]" in green or "[Vorbis]" in `text-secondary` |
| VU meter | Custom `VuMeter` component | Dual vertical bars (L/R). Green/yellow/red gradient. 30Hz timer update. Ballistic smoothing (fast attack 0.8, slow release 0.92) |
| Strip background | Custom painted | `surface-strip` fill with `border-subtle` 1px border. Rounded corners 4px |

**Channel strip dimensions:** 100px wide x full channel area height. 16px horizontal gap between strips.

### Channel Strip Area (`ChannelStripArea`)

| Element | Type | Details |
|---------|------|---------|
| Local strip | `ChannelStrip` | First strip, labeled "Local". Always visible |
| Remote strips | `ChannelStrip` per remote user | Appear/disappear based on connected users. Updated via `UserInfoChangedEvent` |

## Interaction Contracts

### Connection Flow

| State | Connection Bar | Channel Area | Chat Panel |
|-------|---------------|--------------|------------|
| Disconnected | Fields editable, "Connect" button enabled | Dimmed placeholders, "Connect to a server to start jamming", [Browse Servers] button | "Connect to a server to start chatting" |
| Connecting | Fields disabled, "Connect" button shows "Cancel", status dot yellow + "Connecting..." | Same as disconnected (no change until connected) | Same as disconnected |
| License pending | Fields disabled, overlay blocks interaction | License dialog overlay visible | Hidden behind overlay |
| Connected | Fields disabled, "Disconnect" button, status dot green + "Connected", BPM/BPI/users visible | Local + remote channel strips populated with VU meters | Chat active, messages flowing |
| Connection error | Fields re-enabled, "Connect" button, status dot red + error text | Reverts to disconnected state | System message: error description |

### Server Browser Flow

1. User clicks "Browse" -- overlay appears, `RequestServerListCommand` pushed to cmd_queue
2. "Loading servers..." displayed while waiting for `ServerListEvent`
3. Server list populates when event arrives (or error message on failure)
4. Single-click on row: server address copied to connection bar field, overlay stays open
5. Double-click on row: server address copied + `ConnectCommand` pushed, overlay closes (D-14)
6. Close button or Escape key: overlay closes without action

### Chat Flow

1. User types in input field, presses Enter or clicks "Send"
2. Input parsed via `parse_chat_input()` (ported from `src/ui/ui_chat.cpp`)
3. Valid input: `SendChatCommand` pushed to cmd_queue, input cleared, input field refocused
4. Invalid input: system error message displayed locally ("error: invalid command.")
5. Not connected: system error message displayed locally ("error: not connected to a server.")
6. Incoming messages: drained from `chat_queue` in timer callback, appended to message list
7. Auto-scroll: viewport scrolls to bottom on new message UNLESS user has scrolled up (D-16, D-17)
8. Jump-to-bottom: pill appears when scrolled up. Click or new-message-while-at-bottom scrolls down

### VU Meter Behavior

| Property | Value |
|----------|-------|
| Update rate | 30Hz (independent Timer on VuMeter component) |
| Attack coefficient | 0.8 (fast rise to target) |
| Release coefficient | 0.92 (slow decay, ~300ms to half) |
| Color thresholds | 0--70% green, 70--90% yellow, 90--100% red |
| Orientation | Vertical bars, bottom-to-top fill |
| Channel display | Dual bars (L + R) with 4px gap |

## State Management Contract

UI state that must survive editor destruction (stored on Processor, not Editor):

| State | Owner | Type | Reason |
|-------|-------|------|--------|
| Chat history | Processor (`chat_queue` + Processor-owned `std::vector<ChatMessage>`) | `std::vector<ChatMessage>` | Editor recreated when host closes/reopens plugin UI (Pitfall 2) |
| Server list cache | Processor | `std::vector<ServerListEntry>` | Avoid re-fetching when editor reopens |
| Last server address | Processor (APVTS or custom state) | `juce::String` | Persist across sessions |
| Last username | Processor (APVTS or custom state) | `juce::String` | Persist across sessions |
| Selected codec | Processor (APVTS) | `int` (0=FLAC, 1=Vorbis) | Already in APVTS from Phase 1 |
| Scale factor | Processor (custom state) | `float` | Persist user preference |

UI-only state (lives on Editor, recreated fresh):

| State | Type | Reason |
|-------|------|--------|
| Scroll position in chat | `float` | Ephemeral -- scroll to bottom on open is fine |
| Server browser open/closed | `bool` | Always starts closed |
| Password field visible | `bool` | Always starts hidden |
| Auto-scroll enabled | `bool` | Always starts true |

## Event/Command Communication

| Direction | Mechanism | Rate | Items |
|-----------|-----------|------|-------|
| UI -> Run thread | `SpscRing<UiCommand, 256> cmd_queue` (existing) | User-driven (clicks, typing) | ConnectCommand, DisconnectCommand, SendChatCommand, SetEncoderFormatCommand, RequestServerListCommand |
| Run thread -> UI | `SpscRing<UiEvent, 256> evt_queue` (new on Processor) | Event-driven | StatusChangedEvent, UserInfoChangedEvent, TopicChangedEvent, ServerListEvent |
| Run thread -> UI (chat) | `SpscRing<ChatMessage, 128> chat_queue` (new on Processor) | Per-message | ChatMessage structs |
| Run thread -> UI (license) | `atomic<bool> license_pending` + `condition_variable` | Rare (once per connect) | License text string |
| UI polling | Timer at 20Hz in Editor `timerCallback()` | 20Hz | Drains evt_queue + chat_queue, polls cached_status atomic |
