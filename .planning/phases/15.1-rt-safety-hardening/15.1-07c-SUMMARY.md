# Phase 15.1 Plan 07c: DecodeMediaBuffer SPSC Migration Summary

## One-liner

Replaced `DecodeMediaBuffer`'s internal `WDL_Mutex + WDL_Queue` with `SpscRing<DecodeChunk, 32>` plus an audio-thread-owned linear consumer buffer; refcount is now `std::atomic<int>` with `fetch_sub(acq_rel)` semantics — UAF-safe across audio/run thread Release races. External `Read(void*, int)` / `Write(...)` / `Size()` / `AddRef()` / `Release()` API preserved per CONTEXT D-04. **Closes AUDIT CR-12.**

public:
  DecodeMediaBuffer() { refcnt=1; rdpos=0; }
  ~DecodeMediaBuffer() {}

**AFTER** (~110 lines):

```cpp
class DecodeMediaBuffer {
public:
  DecodeMediaBuffer() = default;
  ~DecodeMediaBuffer() = default;

  void AddRef() { m_refcnt.fetch_add(1, std::memory_order_relaxed); }
  void Release() {
    // acq_rel: the last decrement must happen-after all prior writes from
    // either thread, so the destructor sees a consistent state.
    if (m_refcnt.fetch_sub(1, std::memory_order_acq_rel) == 1) delete this;
  }

  // Run/network thread. Splits caller's data into chunk-sized records and
  // pushes them into the SPSC. Returns bytes successfully pushed.
  int Write(const void *buf, int len) {
    if (len <= 0 || !buf) return 0;
    const uint8_t *in = static_cast<const uint8_t*>(buf);
    int remaining = len;
    while (remaining > 0) {
      jamwide::DecodeChunk chunk{};
      chunk.len = std::min(remaining, jamwide::CHUNK_BYTES);
      std::memcpy(chunk.data, in, static_cast<size_t>(chunk.len));
      if (!m_chunks.try_push(chunk)) {
        m_write_drops.fetch_add(1, std::memory_order_relaxed);
        return len - remaining;          // partial: ring full, NINJAM codec absorbs loss
      }
      in += chunk.len; remaining -= chunk.len;
      m_total_written.fetch_add(chunk.len, std::memory_order_relaxed);
    }
    return len;
  }

  // Audio thread. Drains chunks via SPSC into linear consumer buffer; serves
  // up to len bytes. Lock-free; never blocks; never allocates; never enters
  // any kernel mutex (CR-12 closed).
  int Read(void *buf, int len) {
    if (len <= 0 || !buf) return 0;
    int written = 0;
    uint8_t *out = static_cast<uint8_t*>(buf);
    while (written < len) {
      const int avail = m_consumer_buf_len - m_consumer_buf_pos;
      if (avail > 0) {
        const int take = std::min(avail, len - written);
        std::memcpy(out + written, m_consumer_buf.data() + m_consumer_buf_pos, take);
        m_consumer_buf_pos += take; written += take;
        continue;
      }
      auto chunk = m_chunks.try_pop();
      if (!chunk) break;                  // empty -> short-read; caller signals codec
      const int clen = chunk->len;
      if (clen <= 0 || clen > jamwide::CHUNK_BYTES) continue;  // defensive bounds-check
      std::memcpy(m_consumer_buf.data(), chunk->data, clen);
      m_consumer_buf_len = clen;
      m_consumer_buf_pos = 0;
    }
    return written;
  }

  // Run-thread helper. Cumulative bytes written (legacy m_buf.Available()
  // semantics — never compacted). Used by RemoteDownload::startPlaying
  // pre-buffer threshold check.
  int Size() const {
    return static_cast<int>(m_total_written.load(std::memory_order_relaxed));
  }

private:
  jamwide::SpscRing<jamwide::DecodeChunk, 32>     m_chunks;
  std::array<uint8_t, jamwide::CHUNK_BYTES * 2>   m_consumer_buf{};
  int                                             m_consumer_buf_len = 0;
  int                                             m_consumer_buf_pos = 0;
  std::atomic<int>                                m_refcnt{1};
  std::atomic<int64_t>                            m_total_written{0};
  std::atomic<uint64_t>                           m_write_drops{0};
};
```

### Refcount migration

- `int refcnt` → `std::atomic<int> m_refcnt{1}`
- `Release()`: `if (!--refcnt)` → `if (m_refcnt.fetch_sub(1, std::memory_order_acq_rel) == 1)`
- `AddRef()`: `refcnt++` → `m_refcnt.fetch_add(1, std::memory_order_relaxed)`

The `acq_rel` on the decrement is the canonical idiom for refcount-managed-delete: the load-acquire half ensures the destructor sees all prior writes from any other releasing thread; the store-release half publishes any pending writes before we drop the count. Validated by a 1000-trial concurrent Release() race test (`test_atomic_refcnt`) that asserts exactly one delete fires per trial — passes under both Release and TSan with zero races.

### External API contract preserved (CONTEXT D-04)

| Method | Before | After | Caller-visible change |
|--------|--------|-------|------------------------|
| `Read(void*, int) -> int` | mutex-protected | lock-free SPSC drain | None — same signature, same semantics (returns bytes written) |
| `Write(const void*, int)` | `-> void` | `-> int` (bytes pushed) | None at the only existing call site (`RemoteDownload::Write` discards the return). New observability for partial writes on full-ring overflow. |
| `Size() -> int` | `m_buf.Available()` (cumulative — legacy never compacted) | `m_total_written.load()` | None — preserves legacy "cumulative bytes written" semantics so `RemoteDownload::startPlaying` pre-buffer threshold check stays correct |
| `AddRef()` | `refcnt++` | `m_refcnt.fetch_add(1, relaxed)` | None |
| `Release()` | `if (!--refcnt) delete this;` (mutex-protected) | `if (m_refcnt.fetch_sub(1, acq_rel) == 1) delete this;` | None — UAF-safe under concurrent calls (was technically racy before; the legacy mutex was inside the queue, NOT around refcnt) |
| `Avail()` | `m_buf.Available() - rdpos` | **REMOVED** | No callers in the codebase (verified by grep across src/, juce/, tests/) |

## Decisions Made

See frontmatter `key-decisions` for the full list. Summary of the 8 key decisions:

1. **API preservation per CONTEXT D-04.** External `Read(void*, int) -> int` signature unchanged; `Write` extended from `void` to `int` return (strict superset — caller discards); `Size()` preserves legacy cumulative-bytes-written semantics for the run-thread pre-buffer threshold check.

3. **Producer-side overflow drops the chunk + returns short.** NINJAM codec absorbs frame loss. Drop counter (`m_write_drops`) mirrors 15.1-07b's `m_block_queue_drops` observability pattern.

5. **Linear consumer buffer sized 2 * CHUNK_BYTES = 8 KB.** Pattern E from RESEARCH: chunk-granular SPSC handoff with byte-granular Read API; the linear buffer is the impedance match.
6. **Atomic refcount uses `fetch_sub(acq_rel)`.** Standard refcount-managed-delete idiom; ensures destructor happens-after all prior writes from any releasing thread.
7. **spsc_payloads.h UNTOUCHED.** Wave-0 M-9 finality preserved. DecodeChunk + CHUNK_BYTES were already declared at Wave 0 specifically for this plan.

## Deviations from Plan

**None — plan executed exactly as written.**

The plan's prescribed implementation patterns (action 1-7 in Task 1; action 1-5 in Task 2) were followed verbatim with three minor improvements that are within the plan's spirit:

1. **Plan code in Task 1 step 5 had `Release()` not match the prior implementation** — the plan said "Verify the existing Release impl had this shape; if it used a mutex-protected int, this is the migration." The legacy was `if (!--refcnt) delete this;` (NOT mutex-protected; the mutex was around the WDL_Queue, NOT around the refcount itself). Per the plan's instruction, the migration is correct — the refcount becomes atomic and the delete-on-zero contract is preserved with explicit memory ordering. Documented in the class docblock and in test 4.

2. **Plan code in Task 1 step 4 had `Write` returning `int len`** but did not explicitly handle `len <= 0` or `buf == nullptr` defensively. Added defensive guards at the top of Write() and Read() (return 0 immediately) — this matches the legacy WDL_Queue::Add behavior on null/zero-length input and prevents undefined behavior from a malformed caller. No call site in the production codebase passes such inputs today.

3. **Plan code did NOT specify how to preserve `Size()`** — the plan's class definition omits `Size()` entirely. Adding `Size()` as a documented public method that returns `m_total_written.load()` was necessary because `RemoteDownload::startPlaying:4724` calls `m_decbuf->Size() > playtime` to gate the pre-buffer threshold. Removing or breaking this would have been a regression (the audio playback start gate would no longer fire, and remote audio would either start too early or never). This is a faithful preservation of the legacy contract — added test 6 to lock the semantic.

These are not "deviations" in the rules sense (Rules 1-4); they are **plan-faithful expansions** of incomplete plan code. Documented here for traceability.
