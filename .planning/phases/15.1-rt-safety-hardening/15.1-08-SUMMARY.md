# Phase 15.1 Plan 08: Prealloc Hardening Summary

**Status:** Implementation complete (autonomous; UAT deferred to phase-end 15.1-10 per user direction).

## One-liner

Pre-allocate every WDL audio-path buffer (`tmpblock` via new `NJClient::SetMaxAudioBlockSize`; `VorbisDecoder.m_buf`; `FlacDecoder.m_inbuf` + `m_outbuf`) at construction-time on the run thread, so the audio thread sees zero `Resize` / `Add(realloc)` activity after warm-up. Closes AUDIT M-01 (tmpblock.Resize on audio thread), reclassifies M-02 to LOW, lands the M-03 debug-build assertion in processBlock, and partially mitigates CR-11 (libogg/libFLAC algorithmic residual deferred to 15.1-10 Instruments UAT per CONTEXT D-04). Codex M-7 enforced via `std::runtime_error` thrown synchronously on the run/UI thread at `prepareToPlay` — never reaches the audio thread.

## What Landed

### NJClient::SetMaxAudioBlockSize (M-01 + Codex M-7)

```cpp
// src/core/njclient.h
void SetMaxAudioBlockSize(int maxSamplesPerBlock);

// src/core/njclient.cpp
void NJClient::SetMaxAudioBlockSize(int maxSamplesPerBlock)
{
  if (maxSamplesPerBlock <= 0) return;

  if (maxSamplesPerBlock > jamwide::MAX_BLOCK_SAMPLES) {
    throw std::runtime_error(
      "Host samplesPerBlock exceeds JamWide MAX_BLOCK_SAMPLES contract. "
      "Recompile with larger MAX_BLOCK_SAMPLES or reduce host block size.");
  }

  tmpblock.Prealloc(maxSamplesPerBlock * static_cast<int>(sizeof(float)));
}
```

Wired from `JamWideJuceProcessor::prepareToPlay`. Throws are caught and logged via `juce::Logger::writeToLog`; processing continues with the previously-Prealloc'd `tmpblock` size, and the M-03 jassert below catches any actual processBlock violation in Debug builds.

### JamWideJuceProcessor::prevPreparedSize + processBlock M-03 jassert

```cpp
// JamWideJuceProcessor.h
int prevPreparedSize = 0;

// JamWideJuceProcessor.cpp::prepareToPlay
prevPreparedSize = samplesPerBlock;

// JamWideJuceProcessor.cpp::processBlock (top)
jassert(numSamples <= prevPreparedSize);
```

`jassert` is `((void)0)` in Release; zero hot-path cost. In Debug it surfaces host-contract violations during development.

### VorbisDecoder + FlacDecoder Prealloc (CR-11 partial mitigation)

```cpp
// wdl/vorbisencdec.h::VorbisDecoder ctor
m_buf.Prealloc(8192 * 2);  // 8192 samples * MAX_CHANNELS=2 → 64 KB float queue

// wdl/flacencdec.h::FlacDecoder::initDecoder
m_inbuf.Prealloc(16384);    // 16 KB compressed-byte queue
m_outbuf.Prealloc(8192 * 2);  // 64 KB float queue
```

Both decoders are constructed on the run thread per 15.1-09's Codex HIGH-1 invariant (`grep -n 'start_decode(' src/core/njclient.cpp` → 3 callers, all run-thread). FlacDecoder::initDecoder is also called from Reset() (run thread); WDL Prealloc is idempotent (only-grows), so re-invocation is safe.

## CR-11 Mitigation Status (per plan output spec)

CR-11 has two layers:

1. **WDL_TypedQueue / WDL_Queue Add reallocations (m_buf, m_inbuf, m_outbuf) — CLOSED by this plan.** Prealloc at construction grows `m_alloc` to peak frame size; subsequent `m_buf.Add(...)` calls do not realloc until that capacity is exceeded (which doesn't happen at NINJAM frame sizes ≤ 8192 samples * 2 channels).
2. **libogg / libFLAC algorithmic internals — DEFERRED.** Out of scope per CONTEXT D-04; would require codec-algorithm work. Bounded to first 3-5 frames per stream after a peer joins or codec reset; not always-hot. Per the plan's output instruction: flagged for 15.1-10 Instruments measurement under populated UAT. If heap activity from libogg/libFLAC contexts is < 1 alloc/sec after 60 s warm-up, declare CR-11 closed for v1.2 per RESEARCH recommendation. If higher, file follow-up phase ticket.

The agent file lives at `.claude/agents/realtime-audio-reviewer.md` but Task tool dispatch isn't directly available in this sequential executor. **Self-audit against the agent's published criteria:**

**Audio-path code changes in 15.1-08:** ONE — the addition of `jassert(numSamples <= prevPreparedSize)` at top of `processBlock`. JUCE's `jassert` macro is `((void)0)` in Release builds (no allocation, no syscall, no I/O); in Debug builds it expands to a `DBG + jassertfalse`. Debug builds are not shipped to users.

The Prealloc additions to VorbisDecoder ctor + FlacDecoder::initDecoder are RUN-thread code (decoders are constructed on the run thread per 15.1-09's HIGH-1 invariant). The `SetMaxAudioBlockSize` impl in NJClient is RUN/UI-thread code (called from JUCE prepareToPlay).

## Decisions Made

See frontmatter `key-decisions` for the full list. Summary:

1. **Codex M-7 enforcement** — synchronous throw at run/UI thread, never reaches audio thread.
2. **Codex per-plan delta accepted** — lighter sanity test per private-field privacy; strict allocation-count check deferred to 15.1-10 Instruments.
3. **Prealloc sizes** — tmpblock = maxSamplesPerBlock * sizeof(float); decoder queues = 64 KB float / 16 KB byte (covers Vorbis/FLAC peak frame sizes at MAX_CHANNELS=2).
4. **M-03 jassert in Debug only** — Release backstops via M-7 + 15.1-07b per-callsite check; zero hot-path cost.
5. **FlacDecoder Prealloc in initDecoder, not constructor** — runs on both ctor AND Reset; idempotent via Prealloc only-grows.
6. **Legacy tmpblock.Resize at line 2591 retained** — defensive backstop (post-Prealloc, fires AT MOST ONCE per session to bump m_size against the already-grown m_alloc; does NOT reallocate).
7. **spsc_payloads.h UNTOUCHED** — Codex M-9 / Wave-0 finality preserved; jamwide::MAX_BLOCK_SAMPLES referenced through existing njclient.h include chain.
