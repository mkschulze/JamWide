2. Implement `deriveRoomHash` (per D-05, D-06, D-07 -- RESEARCH.md Open Question 1 recommends using `#password=DERIVED_PASSWORD` approach for VDO.Ninja compatibility):
```cpp
juce::String VideoCompanion::deriveRoomHash(const juce::String& password,
                                            const juce::String& roomId)
{
    if (password.isEmpty()) return {};  // D-07: no hash for public rooms

    // D-06: SHA-256(ninjam_password + ":" + room_id)
    // Use as VDO.Ninja password (not hash) per RESEARCH.md Open Question 1:
    // VDO.Ninja sees a password and handles its own internal hashing.
    // All JamWide users with same NINJAM password derive the same VDO.Ninja password.
    juce::String input = password + ":" + roomId;
    juce::SHA256 sha(input.toUTF8());
    // 16 hex chars = 8 bytes of entropy, sufficient for room-level security
    return sha.toHexString().substring(0, 16);
}
```

```cpp
bool VideoCompanion::launchCompanion(const juce::String& serverAddr,
                                     const juce::String& username,
                                     const juce::String& password)
{
    currentRoom_ = deriveRoomId(serverAddr, password);
    currentPush_ = sanitizeUsername(username);
    currentPassword_ = password;  // Store for hash derivation

    if (!startWebSocketServer())
    {
        DBG("VideoCompanion: Failed to start WebSocket server on port " + juce::String(wsPort_));
        return false;
    }

    auto hashFragment = deriveRoomHash(password, currentRoom_);
    currentCompanionUrl_ = buildCompanionUrl(currentRoom_, currentPush_, wsPort_, hashFragment);
    juce::URL(currentCompanionUrl_).launchInDefaultBrowser();
    active_.store(true, std::memory_order_relaxed);

    // Compute initial buffer delay from current BPM/BPI (Pitfall 6: cache for late-connecting WS clients)
    float bpm = processor_.uiSnapshot.bpm.load(std::memory_order_relaxed);
    int bpi = processor_.uiSnapshot.bpi.load(std::memory_order_relaxed);
    if (bpm > 0.0f && bpi > 0)
        cachedDelayMs_ = static_cast<int>((60.0 / static_cast<double>(bpm)) * bpi * 1000.0);

    return true;
}
```

5. Implement `broadcastBufferDelay` (per D-01, D-03, D-04):

```cpp
void VideoCompanion::broadcastBufferDelay(float bpm, int bpi)
{
    // D-02: Called on BPM/BPI change only, not on every beat
    if (bpm <= 0.0f || bpi <= 0) return;
    if (!isActive()) return;  // Pitfall 6: guard if video not active

    // D-03: delay_ms = (60.0 / bpm) * bpi * 1000
    cachedDelayMs_ = static_cast<int>((60.0 / static_cast<double>(bpm)) * bpi * 1000.0);

    // D-04: {"type":"bufferDelay","delayMs":N}
    juce::String json = "{\"type\":\"bufferDelay\",\"delayMs\":"
                        + juce::String(cachedDelayMs_) + "}";

    std::lock_guard<std::mutex> lock(wsMutex_);
    if (!wsServer_) return;
    auto clients = wsServer_->getClients();
    for (auto& client : clients)
        client->send(json.toStdString());
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Plugin -> Companion (WebSocket) | Plugin sends JSON to companion page over localhost WebSocket. Messages contain room config and buffer delay (no secrets). |
| Plugin -> Browser (URL) | Companion URL contains room hash in fragment portion. Fragment never sent to server. |
| NINJAM password -> SHA-256 | Password is hashed before any network transmission. Only the derived hash appears in URLs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Information Disclosure | VideoCompanion::buildCompanionUrl | mitigate | Hash appended as URL fragment (#password=), never query parameter (?). Fragment is browser-only per RFC 3986 section 3.5. Verified in code: url += "#password=" not "?password=". |
| T-12-02 | Information Disclosure | VideoCompanion::sendConfigToClient | accept | Config WebSocket message intentionally excludes password (carried forward from Phase 11 D-14). Buffer delay message contains only integer milliseconds -- no sensitive data. |
| T-12-03 | Spoofing | deriveRoomHash | accept | SHA-256 with password+roomId salt makes hash unpredictable without knowing password. 16 hex chars = 64 bits entropy. An attacker who guesses the NINJAM password already has session access, so hash guessing adds no additional attack surface. |
| T-12-04 | Tampering | WebSocket bufferDelay message | accept | Buffer delay is a non-critical visual parameter. Tampering would only affect video buffering timing, not audio or security. WebSocket is localhost-only (127.0.0.1 binding from Phase 11 T-11-01). |
</threat_model>
