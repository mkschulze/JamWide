Create VbFader as a fully custom juce::Component (NOT a juce::Slider subclass -- per research anti-pattern). This is the Voicemeeter Banana style big vertical fader per D-01.

**VbFader.h:**
- Class inherits `juce::Component`
- Public API:
  - `void setValue(float newValue)` -- linear 0.0 to 2.0
  - `float getValue() const`
  - `void setDefaultValue(float val)` -- double-click reset target
  - `std::function<void(float)> onValueChanged` -- callback for non-APVTS use (remote channels)
  - `void attachToParameter(juce::RangedAudioParameter& param, juce::UndoManager* um = nullptr)` -- for APVTS-bound local channels
  - `void adjustByDb(float dbDelta)` -- used by ChannelStrip scroll wheel forwarding (D-02)
- Private members:
  - `float value_ = 1.0f` (0 dB default)
  - `float defaultValue_ = 1.0f`
  - `int dragStartY_ = 0`
  - `float dragStartValue_ = 0.0f`
  - `std::unique_ptr<juce::ParameterAttachment> attachment_`
  - Constants: `kTrackWidth = 10`, `kThumbDiameter = 44`, `kMinLinear = 0.0f`, `kMaxLinear = 2.0f`
- Override: paint, mouseDown, mouseDrag, mouseUp, mouseDoubleClick, mouseWheelMove
- JUCE_DECLARE_NON_COPYABLE_WITH_LEAK_DETECTOR

**VbFader.cpp:**

`paint()` implementation per UI-SPEC:
1. Track: 10px wide, centered horizontally in component bounds, filled with `kVuBackground` (0xff0A0D18), 2px rounded corners
2. Green fill: from bottom of track up to thumb Y position, color `kAccentConnect` (0xff40E070), same 10px width
3. dB scale ticks: 1px marks to the right of the track at +4px offset, at Y positions corresponding to linear values 2.0 (+6), 1.0 (0), 0.5012 (-6), 0.1259 (-18), 0.0 (-inf). Labels in 9px Regular font, `kTextSecondary` (0xff8888AA)
4. Thumb: 44px diameter circle centered on track at thumb Y position. Fill `kSurfaceStrip` (0xff2A2D48), border 2px `kAccentConnect` (0xff40E070)
5. dB text ON thumb: centered, 11px Bold font, `kTextPrimary` (0xffE0E0E0). Format per D-05: one decimal place with sign. Use `formatDb()` helper.

`formatDb(float linear)` helper:
- If `linear < 0.001f` return `"-inf"`
- `float db = 20.0f * std::log10(linear)`
- Clamp to 6.0f max
- Format: `db >= 0 ? "+" + String(db, 1) : String(db, 1)`

`valueToY(float val)` -- maps linear 0.0-2.0 to pixel Y coordinate:
- Use power-curve mapping for better dB resolution in low/mid range per research Open Question 1
- `float norm = val / kMaxLinear` (0.0 to 1.0)
- Apply inverse power curve: `float curveNorm = std::pow(norm, 1.0f / 2.5f)` -- this gives more fader travel to the lower dB range
- `float topPad = kThumbDiameter / 2.0f` (so thumb doesn't clip at top)
- `float botPad = kThumbDiameter / 2.0f` (so thumb doesn't clip at bottom)
- `float travel = getHeight() - topPad - botPad`
- Return `topPad + travel * (1.0f - curveNorm)` -- top = max, bottom = min

`yToValue(int y)` -- inverse of valueToY:
- `float topPad = kThumbDiameter / 2.0f`
- `float botPad = kThumbDiameter / 2.0f`
- `float travel = getHeight() - topPad - botPad`
- `float curveNorm = 1.0f - (y - topPad) / travel`
- Clamp curveNorm to [0, 1]
- `float norm = std::pow(curveNorm, 2.5f)` -- apply power curve
- Return `juce::jlimit(kMinLinear, kMaxLinear, norm * kMaxLinear)`

`mouseDown()`:
- Store `dragStartY_ = e.y` and `dragStartValue_ = value_`
- If attachment_: `attachment_->beginGesture()`

`mouseDrag()`:
- Compute new value from absolute Y position: `float newVal = yToValue(e.y)`
- Snap near-zero: if `newVal < 0.001f` set to 0.0f (per pitfall 1)
- Call `setValue(newVal)`

`mouseUp()`:
- If attachment_: `attachment_->endGesture()`

`mouseDoubleClick()` per D-03:
- `setValue(defaultValue_)` (resets to 0 dB = 1.0 linear)
- No animation in this implementation (100ms animation deferred -- per UI-SPEC it is optional for Phase 5)

`mouseWheelMove()` per D-02:
- Convert current value to dB: `float currentDb = (value_ <= 0.0001f) ? -100.0f : 20.0f * std::log10(value_)`
- Add `wheel.deltaY * 0.5f` (0.5 dB per scroll step)
- Clamp to [-100, 6] dB range
- Convert back: `float newLinear = (newDb <= -100.0f) ? 0.0f : std::pow(10.0f, newDb / 20.0f)`
- `setValue(juce::jlimit(kMinLinear, kMaxLinear, newLinear))`
- Do NOT call `Component::mouseWheelMove()` -- consume the event (per pitfall 4)

`adjustByDb(float dbDelta)`:
- Same logic as mouseWheelMove but with explicit dB delta parameter
- Used by ChannelStrip to forward scroll events (D-02)

`setValue(float newValue)`:
- Clamp to [kMinLinear, kMaxLinear]
- If attachment_: `attachment_->setValueAsPartOfGesture(newValue)` (only during drag gesture)
- Else if no gesture active and attachment_: `attachment_->setValueAsCompleteGesture(newValue)` (for double-click, scroll)
- Store `value_ = newValue`
- Call `onValueChanged(value_)` if callback set
- `repaint()`

`attachToParameter()`:
- Create `attachment_ = std::make_unique<juce::ParameterAttachment>(param, [this](float v) { value_ = v; repaint(); }, um)`
- Call `attachment_->sendInitialUpdate()` to sync initial value
