# Plan 08-01: Wire Remote Codec Badge and Surface Error Messages

## Objective

Close the two remaining integration gaps from the v1.0 milestone audit: (1) populate remote channel codec badges so users can see what codec each participant is using, and (2) surface server error messages in the status label instead of hardcoded strings.

Note: CODEC-05 (codec selector default) was already fixed in this session — the UI and NJClient now both default to Vorbis.

## Task 1: Wire Remote Codec Badge (CODEC-03)

### Problem
`ChannelStripArea.cpp:380` passes empty `codecStr` to `ChannelStrip.configure()`. `GetUserChannelCodec()` exists on NJClient but is never called. Users can't see what codec remote participants are using.

### Changes

**File: `src/core/njclient.h`** — Add `codec_fourcc` to `RemoteChannelInfo`
- Add `unsigned int codec_fourcc = 0;` field to `RemoteChannelInfo` struct (after `out_chan_index`)

**File: `src/core/njclient.cpp`** — Populate codec_fourcc in `GetRemoteUsersSnapshot()`
- In the snapshot loop where `RemoteChannelInfo` is populated, add:
  `ch_info.codec_fourcc = chan->codec_fourcc;`
- This reads the FOURCC that was set when the remote channel's audio was decoded

**File: `juce/ui/ChannelStripArea.cpp`** — Read codec from snapshot and pass to strip
- In `refreshFromUsers()` single-channel path (~line 380):
  Replace `juce::String codecStr;` with codec lookup from the channel:
  ```cpp
  juce::String codecStr;
  unsigned int fourcc = user.channels[0].codec_fourcc;
  if (fourcc == 0x43414C46) codecStr = "FLAC";      // 'FLAC'
  else if (fourcc == 0x7647474F) codecStr = "Vorbis"; // 'OGGv'
  ```
- Apply same pattern in multi-channel path for child strips

### Key Links
- `src/core/njclient.h:175` RemoteChannelInfo struct
- `src/core/njclient.cpp` GetRemoteUsersSnapshot() population loop
- `juce/ui/ChannelStripArea.cpp:380` single-channel strip configure
- `juce/ui/ChannelStrip.h` configure() already accepts codecStr parameter

### Verification
- Remote channel strips show "FLAC" or "Vorbis" badge text
- Badge updates when remote user changes codec

## Task 2: Surface Error Messages (UI-01, UI-03)

### Problem
`StatusChangedEvent.error_msg` is drained and discarded in `JamWideJuceEditor::drainEvents()`. `ConnectionBar::updateStatus()` uses hardcoded strings ("Connection failed", "Invalid credentials") instead of the actual server error.

### Changes

**File: `juce/JamWideJuceProcessor.h`** — Add error message storage
- Add `juce::String lastErrorMsg;` member (written by editor drain, read by ConnectionBar)

**File: `juce/JamWideJuceEditor.cpp`** — Store error_msg from StatusChangedEvent
- In `drainEvents()`, replace the empty `StatusChangedEvent` handler (~line 232-235):
  ```cpp
  if constexpr (std::is_same_v<T, jamwide::StatusChangedEvent>)
  {
      if (!e.error_msg.empty())
          processorRef.lastErrorMsg = juce::String(e.error_msg);
  }
  ```

**File: `juce/ui/ConnectionBar.cpp`** — Use stored error message
- In `updateStatus()` for `NJC_STATUS_CANTCONNECT` and `NJC_STATUS_INVALIDAUTH` cases (~lines 380-386):
  Check `processorRef.lastErrorMsg` and use it if non-empty, otherwise fall back to hardcoded string:
  ```cpp
  case NJClient::NJC_STATUS_CANTCONNECT:
  {
      auto msg = processorRef.lastErrorMsg.isNotEmpty()
          ? processorRef.lastErrorMsg : juce::String("Connection failed");
      statusLabel.setText(msg, juce::dontSendNotification);
      ...
  }
  ```
- Clear `lastErrorMsg` on successful connect (NJC_STATUS_OK case)

### Key Links
- `src/threading/ui_event.h:31-34` StatusChangedEvent with error_msg field
- `juce/JamWideJuceEditor.cpp:232-235` current no-op drain handler
- `juce/ui/ConnectionBar.cpp:380-386` hardcoded status strings
- `juce/NinjamRunThread.cpp:268-276` where error_msg is populated from GetErrorStr()

### Verification
- Connection errors show server-provided message instead of generic text
- Successful connect clears any previous error message
