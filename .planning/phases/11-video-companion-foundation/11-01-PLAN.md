**Step 2: Create juce/video/VideoCompanion.h — FROZEN PUBLIC INTERFACE.**

This header defines the contract that Plan 03 depends on. Do NOT change the public API after this plan.

Addresses review concern: "VideoCompanion interface not frozen" (HIGH). Plan 03 depends on exactly this interface.

```cpp
#pragma once
#include <JuceHeader.h>
#include <ixwebsocket/IXWebSocketServer.h>
#include <atomic>
#include <memory>
#include <string>
#include <vector>
#include <set>
#include <mutex>

class JamWideJuceProcessor;

// Forward declare to avoid pulling njclient.h into header
namespace NJClient_ns {}  // placeholder — use full type in .cpp
struct NJClientRemoteUserInfo;  // Not needed — we use the real struct via include in .cpp

namespace jamwide {

/// VideoCompanion: Owns WebSocket server, room ID derivation, username sanitization,
/// and browser launch for the video companion page.
///
/// FROZEN PUBLIC INTERFACE — Plan 03 depends on exactly these public methods.
/// Do not add, remove, or change signatures without updating Plan 03.
///
/// Thread safety:
/// - launchCompanion(): call from message thread only
/// - relaunchBrowser(): call from message thread only (re-click while active, per D-04)
/// - onRosterChanged(): call from ANY thread (marshals to message thread via callAsync)
/// - isActive(): lock-free atomic read, safe from any thread
/// - deactivate(): call from message thread only
///
/// Key definitions:
/// - push: The local user's sanitized username used as their VDO.Ninja stream ID.
///   Derived by sanitizeUsername(username). Example: "Dave@guitar" -> "Daveguitar"
/// - streamId: Each roster entry's collision-resolved sanitized name, used to identify
///   their VDO.Ninja stream. For roster users, collisions are resolved with "_N" suffix
///   using deterministic user-index-based ordering (not join order).
/// - Room ID: SHA-1 hash of (serverAddr + ":" + password) or (serverAddr + ":jamwide-public")
///   for password-less servers. Prefix "jw-", 16 hex chars. D-16/D-17 deliberately include
///   password so private servers get unique rooms and public servers share a deterministic room.
class VideoCompanion
{
public:
    explicit VideoCompanion(JamWideJuceProcessor& processor);
    ~VideoCompanion();

    /// Called by message thread when user clicks Video button (after privacy modal accepted).
    /// serverAddr: "host:port", username: local user's name, password: session password (may be empty).
    /// Starts WebSocket server if not already running, builds companion URL, opens browser.
    /// Returns true if launch succeeded, false if WebSocket server failed to start.
    bool launchCompanion(const juce::String& serverAddr,
                         const juce::String& username,
                         const juce::String& password);

    /// Called by message thread on re-click while active (D-04). Re-opens browser only.
    /// Does NOT restart WebSocket server. Does NOT show modal again.
    void relaunchBrowser();

    /// Called from ANY thread (run thread, message thread) when roster changes.
    /// Marshals to message thread via callAsync before accessing wsServer_.
    /// Addresses review concern: "Thread safety for roster -> WebSocket" (HIGH).
    /// Uses same callAsync + alive flag pattern as OscServer.
    void onRosterChanged(const std::vector<NJClient::RemoteUserInfo>& users);

    bool isActive() const { return active_.load(std::memory_order_relaxed); }

    /// Reset state on disconnect (button goes inactive, WS server stops).
    /// Call from message thread only.
    void deactivate();

    static constexpr int kDefaultWsPort = 7170;

private:
    bool startWebSocketServer();
    void stopWebSocketServer();

    juce::String deriveRoomId(const juce::String& serverAddr,
                              const juce::String& password);
    static juce::String sanitizeUsername(const juce::String& name);
    static juce::String resolveCollision(const juce::String& sanitized,
                                         int userIdx,
                                         std::set<juce::String>& assigned);

    juce::String buildCompanionUrl(const juce::String& roomId,
                                   const juce::String& pushId,
                                   int wsPort);

    void sendConfigToClient(ix::WebSocket& client);
    void broadcastRoster(const std::vector<NJClient::RemoteUserInfo>& users);

    JamWideJuceProcessor& processor_;
    std::atomic<bool> active_{false};
    int wsPort_ = kDefaultWsPort;

    std::unique_ptr<ix::WebSocketServer> wsServer_;
    std::mutex wsMutex_;  // Guards wsServer_ start/stop and broadcast

    // Current session config (set on launch, sent to each connecting WS client)
    juce::String currentRoom_;
    juce::String currentPush_;
    juce::String currentCompanionUrl_;

    // callAsync UAF safety — same pattern as OscServer
    std::shared_ptr<std::atomic<bool>> alive_ = std::make_shared<std::atomic<bool>>(true);

    JUCE_DECLARE_NON_COPYABLE_WITH_LEAK_DETECTOR(VideoCompanion)
};

}  // namespace jamwide
```

**Step 3: Create juce/video/VideoCompanion.cpp.**

Include headers:
```cpp
#include "VideoCompanion.h"
```

Implement constructor:
- Store processor reference.
- No other initialization needed.

Implement destructor:
- Set `alive_->store(false, std::memory_order_release)` — prevents pending callAsync lambdas from accessing dead object (same pattern as OscServer).
- Call `stopWebSocketServer()`.

Implement `deriveRoomId(serverAddr, password)` per D-16, D-17:
- Build input string: if password is empty, use `serverAddr + ":jamwide-public"` (D-17 fixed salt). If password is non-empty, use `serverAddr + ":" + password` (D-16).
- NOTE: Including password is BY DESIGN (addresses review concern #1). D-16 says "Room ID derived from NINJAM server address + session password. Private servers with passwords get unique rooms." This means all users on the same server WITH THE SAME PASSWORD will get the same room. Users with different passwords on the same server get different rooms, which is correct — they are in different NINJAM sessions.
- Create WDL_SHA1 instance.
- Call `sha.add(utf8.getAddress(), static_cast<int>(utf8.sizeInBytes() - 1))` where utf8 = input.toUTF8().
- Call `sha.result(hash)` with `unsigned char hash[WDL_SHA1SIZE]`.
- Convert first 8 bytes to 16 hex chars: loop i=0..7, append `juce::String::toHexString(hash[i]).paddedLeft('0', 2)`.
- Return `"jw-" + hexString` (e.g., "jw-a1b2c3d4e5f6g7h8", 19 chars, valid for VDO.Ninja room name constraint of 1-49 alphanumeric chars + hyphens).

Implement `sanitizeUsername(name)` per D-22:
- Iterate each character of name.
- Keep only characters matching `juce::CharacterFunctions::isLetterOrDigit(c) || c == '_'`.
- If result is empty, return `"user"`.

Implement `resolveCollision(sanitized, userIdx, assigned)` per D-23:
- Addresses review concern #10: "Username collision instability — make deterministic from stable user identity."
- If sanitized is NOT in the assigned set, insert it and return it.
- Otherwise, build candidate = `sanitized + "_" + juce::String(userIdx)`. This uses the NINJAM user index (stable within a roster snapshot) rather than join order, making collision resolution deterministic across roster updates.
- If candidate is also in assigned (unlikely but handle it), try `sanitized + "_" + juce::String(userIdx) + "b"`, `"c"`, etc.
- Insert the unique result into assigned and return it.

Implement `launchCompanion(serverAddr, username, password)`:
1. `currentRoom_ = deriveRoomId(serverAddr, password)`
2. `currentPush_ = sanitizeUsername(username)` — this is the local user's push/stream ID (addresses review concern #2: "push = sanitized username")
3. Start WebSocket server: `bool ok = startWebSocketServer()`
4. If `!ok`: log error "VideoCompanion: Failed to start WebSocket server on port {wsPort_}", return false. (Addresses review concern #8: port bind failure handled gracefully.)
5. Build URL: `currentCompanionUrl_ = buildCompanionUrl(currentRoom_, currentPush_, wsPort_)`
6. Launch browser: `juce::URL(currentCompanionUrl_).launchInDefaultBrowser()`
7. Set `active_.store(true, std::memory_order_relaxed)`
8. Return true.

Implement `relaunchBrowser()` per D-04:
- Addresses review concern #12: re-click while active just re-opens browser, does NOT restart server or create duplicate tabs beyond browser behavior.
- If `!isActive()` return (safety check).
- Call `juce::URL(currentCompanionUrl_).launchInDefaultBrowser()`.
- No server restart, no state change.

Implement `buildCompanionUrl(roomId, pushId, wsPort)` per D-18, D-20:
- Return `"https://video.jamwide.app/?room=" + juce::URL::addEscapeChars(roomId, true) + "&push=" + juce::URL::addEscapeChars(pushId, true) + "&wsPort=" + juce::String(wsPort)`

Implement `startWebSocketServer()`:
- Lock wsMutex_. If wsServer_ already exists, return true (idempotent — addresses review concern #12).
- Create: `wsServer_ = std::make_unique<ix::WebSocketServer>(wsPort_, "127.0.0.1")` — CRITICAL: bind to 127.0.0.1, NOT "localhost" or "0.0.0.0" (Research pitfall 1 and 7).
- Set callback via `wsServer_->setOnClientMessageCallback(...)`:
  - On `ix::WebSocketMessageType::Open`: call `sendConfigToClient(webSocket)`
  - On `ix::WebSocketMessageType::Close`: no-op (logging optional)
- Call `auto res = wsServer_->listen()`.
- If `!res.first`: (addresses review concern #8: port bind failure)
  - Log error: `DBG("VideoCompanion: WebSocket listen failed on port " << wsPort_ << ": " << res.second);`
  - Reset wsServer_.
  - Return false.
- Call `wsServer_->start()` (non-blocking, runs on IXWebSocket's own thread).
- Return true.

Implement `stopWebSocketServer()`:
- Lock wsMutex_. If wsServer_ is null, return.
- Call `wsServer_->stop()`, then `wsServer_->wait()`, then `wsServer_.reset()`.

Implement `sendConfigToClient(client)` per D-13, D-14:
- Build JSON string manually (per Research "Don't Hand-Roll" guidance — only 2 flat message types, manual construction is fine):
  ```
  {"type":"config","room":"<currentRoom_>","push":"<currentPush_>","noaudio":true,"wsPort":<wsPort_>}
  ```
- Escape any quotes in currentRoom_ and currentPush_ values (replace `"` with `\"`).
- Use `client.send(json.toStdString())`.

Implement `onRosterChanged(users)` per D-14:
- Addresses review concern #4: "Thread safety for roster -> WebSocket" (HIGH).
- This method can be called from ANY thread (run thread calls it on roster change).
- Copy the users vector by value into a local copy.
- Use callAsync with alive_ flag to marshal to message thread:
  ```cpp
  auto usersCopy = users;  // copy for lambda capture
  auto aliveFlag = alive_;
  juce::MessageManager::callAsync([this, aliveFlag, usersCopy = std::move(usersCopy)]() {
      if (!aliveFlag->load(std::memory_order_acquire)) return;
      broadcastRoster(usersCopy);
  });
  ```

Implement `broadcastRoster(users)` per D-14:
- This runs on message thread (marshalled from onRosterChanged).
- Lock wsMutex_. If wsServer_ is null, return.
- Build JSON: start with `{"type":"roster","users":[`
- Create a local `std::set<juce::String> assigned` for collision tracking.
- For each user at index i:
  - `auto sanitized = sanitizeUsername(juce::String(users[i].name))`
  - `auto streamId = resolveCollision(sanitized, i, assigned)` — uses user index for deterministic collision resolution (addresses review concern #10)
  - Append: `{"idx":<i>,"name":"<escaped raw name>","streamId":"<streamId>"}`
  - Escape raw name: replace `"` with `\"` and `\` with `\\` in raw name string.
  - Add comma between entries (not after last).
- Close with `]}`.
- Get connected clients from server and broadcast. Use `for (auto& client : wsServer_->getClients())` and call `client->send(json.toStdString())` on each.

Implement `deactivate()`:
- Set `active_.store(false, std::memory_order_relaxed)`
- `stopWebSocketServer()`
- Clear `currentRoom_`, `currentPush_`, and `currentCompanionUrl_`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Plugin -> Companion (WebSocket) | Plugin sends config/roster JSON over ws://127.0.0.1 to companion page |
| Companion -> VDO.Ninja | Companion page loads VDO.Ninja iframe with room/push params |
| Run Thread -> Message Thread | Roster updates marshalled via callAsync (not direct WS access) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Spoofing | WebSocket server | mitigate | Bind to 127.0.0.1 only (not 0.0.0.0); only local processes can connect. No remote access possible. Enforced in VideoCompanion::startWebSocketServer(). |
| T-11-02 | Information Disclosure | Room ID derivation | accept | Room ID is SHA-1 hash of server+password, not reversible to password. Public rooms use fixed salt. Room IDs are not secret -- VDO.Ninja rooms are inherently discoverable by design. |
| T-11-03 | Denial of Service | WebSocket port 7170 | mitigate | On port bind failure: log error, return false from startWebSocketServer(), caller shows user feedback. No crash. Single-instance limitation documented. |
| T-11-04 | Tampering | WebSocket messages | accept | Localhost-only communication. Any process that can connect to 127.0.0.1:7170 already has full local access. No authentication needed for local IPC. |
| T-11-05 | Information Disclosure | Username in push= URL param | accept | NINJAM usernames are already public in the session. Exposing them in VDO.Ninja push IDs adds no new information leak. |
| T-11-06 | Tampering | Run thread -> WS broadcast | mitigate | onRosterChanged uses callAsync to marshal to message thread before accessing wsServer_. Prevents data races. Uses alive_ flag for UAF safety (same pattern as OscServer). |
</threat_model>
