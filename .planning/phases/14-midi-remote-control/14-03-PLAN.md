<objective>
Wire OSC and UI remote-user change paths through APVTS as the single source of truth, eliminating direct cmd_queue dispatch for remote group controls. Currently, OSC and UI change remote user parameters via cmd_queue directly, which (a) bypasses APVTS so MIDI controllers and DAW automation show stale values, and (b) creates double dispatch risk when MidiMapper's timerCallback also syncs APVTS to cmd_queue.

Purpose: This plan makes MidiMapper::timerCallback the SOLE path from APVTS to NJClient for remote group controls. OscServer and ChannelStripArea update APVTS ONLY (no cmd_queue). This eliminates double dispatch entirely (addresses review concern #2) and ensures all change sources (MIDI, OSC, UI, DAW automation) are reflected in APVTS for correct feedback to all controllers.

Output: Updated OscServer.cpp and ChannelStripArea.cpp that write to APVTS remote params only (removing cmd_queue direct dispatch for group controls), with MIDI and OSC echo suppression.
</objective>
