# Phase 15.1 Plan 04: SPSC Infrastructure Summary

**One-liner:** Land Wave-0 SPSC infrastructure — finalized payload shapes (Codex M-9), MAX_BLOCK_SAMPLES contract (M-7), no raw-pointer escape hatches (HIGH-2), deferred-free transport capacity constants (HIGH-3), `--tsan` build target, and a roundtrip + concurrent-stress test harness that runs clean under ThreadSanitizer.

## What Was Built

### `src/threading/spsc_payloads.h` (NEW, 305 lines)

Single header gathering ALL Wave-0 payload definitions. After this plan lands, no subsequent sub-plan (15.1-05 / 06 / 07a / 07b / 07c / 08 / 09) modifies this file — that is the Codex M-9 stability claim. Every variant + POD has a `static_assert(std::is_trivially_copyable_v<T>)` to lock the SPSC element-type contract at compile time.

| Group | Definition | Consumed by | Notes |
|---|---|---|---|
| **(a) RemoteUserUpdate** | variant of 6 PODs: PeerAddedUpdate / PeerRemovedUpdate / PeerChannelMaskUpdate / PeerVolPanUpdate / PeerNextDsUpdate / PeerCodecSwapUpdate | 15.1-07a | PeerAddedUpdate carries `user_index` by value — no `RemoteUser*` field (HIGH-2). PeerNextDsUpdate carries `DecodeState*` ownership-transfer only. |
| **(b) LocalChannelUpdate** | variant of 4 PODs: LocalChannelAddedUpdate / LocalChannelRemovedUpdate / LocalChannelInfoUpdate / LocalChannelMonitoringUpdate | 15.1-06 | LocalChannelAddedUpdate is FULL (channel/srcch/bitrate/bcast/outch/flags + mute/solo/volume/pan) — Codex M-9 finalization, 15.1-06 does not extend it. |
| **(c) BlockRecord** | POD with inline `float samples[MAX_BLOCK_SAMPLES * MAX_BLOCK_CHANNELS]` (16 KB) | 15.1-07b | sizeof bound asserted: ≤ 20 + 16384 + 16 padding = 16420 bytes. |
| **(d) DecodeChunk** | POD with inline `uint8_t data[CHUNK_BYTES]` (4 KB) | 15.1-07c | Replaces DecodeMediaBuffer mutex+queue. |
| **(e) DEFERRED_DELETE_CAPACITY** | constant = 256 | 15.1-05 | DecodeState* deferred delete from audio thread. |
| **(f) DecodeArmRequest** | POD with slot/channel/slot_idx/fourcc/srate/nch/guid[16] | 15.1-09 | **M-9 finalization — included at Wave 0 so 15.1-09 doesn't mutate this header.** |
| **(g) HIGH-3 capacities** | REMOTE_USER_DEFERRED_DELETE_CAPACITY=64, LOCAL_CHANNEL_DEFERRED_DELETE_CAPACITY=32 | 15.1-06 / 15.1-07a | Lifetime contract documented inline; generation-gate enforcement lives in the downstream plans. |

## Capacity Decisions Summary (per 15.1-RESEARCH § SPSC Primitive Inventory)

| Constant | Value | Rationale |
|---|---|---|
| MAX_BLOCK_SAMPLES | 2048 | Largest reasonable host samplesPerBlock; covers JUCE prepareToPlay contracts up to ~46ms at 48kHz |
| MAX_BLOCK_CHANNELS | 2 | Stereo |
| CHUNK_BYTES | 4096 | NINJAM packets typically ≤ 8KB; 4KB chunked |
| DEFERRED_DELETE_CAPACITY | 256 | Worst-case interval-boundary burst (8 peers × 32 channels = 256) absorbed in single drain miss |
| ARM_REQUEST_CAPACITY | 256 | Sessionmode rearm; sessionmode is rare-event but bursty |
| REMOTE_USER_DEFERRED_DELETE_CAPACITY | 64 | Peer churn is run-thread-paced (rare); 64 is generous |
| LOCAL_CHANNEL_DEFERRED_DELETE_CAPACITY | 32 | Matches MAX_LOCAL_CHANNELS |
| RemoteUserUpdate ring (per-NJClient) | 64 | State changes are run-thread-paced; declared in 15.1-07a |
| LocalChannelUpdate ring (per-NJClient) | 32 | Bounded by MAX_LOCAL_CHANNELS; declared in 15.1-06 |
| BlockRecord ring (per local channel) | 16 | ~320 ms headroom at 50 ms blocks; declared in 15.1-07b |
| DecodeChunk ring (per active stream) | 32 | ~128KB per stream; declared in 15.1-07c |
