## Critical Anti-Patterns

| Severity | Pattern | Why it matters |
|----------|---------|----------------|
| blocking | DO NOT introduce locks, heap allocations, or I/O on the audio path | The whole phase exists to remove these — re-adding any one negates the work. Use the `realtime-audio-reviewer` agent at `.claude/agents/realtime-audio-reviewer.md` to verify any audio-path code change. |
| blocking | DO NOT call `delete` on `DecodeState*` from any function reachable from `processBlock`/`AudioProc`/`process_samples`/`mixInChannel`/`on_new_interval` | 15.1-05 just converted all 7 audit-cited sites to deferred-delete. New code touching these functions must use the same pattern, not direct `delete`. |
| blocking | DO NOT introduce `lc_ptr` / `user_ptr` raw back-pointers in payload definitions | Codex HIGH-2 — payloads must NOT carry pointers back into run-thread-owned canonical objects. 15.1-04 has a grep gate that fails the build if this regresses. |
| blocking | DO NOT block the audio thread waiting for the run-thread to drain a queue | Use `tryPush` (non-blocking, returns false on full); increment overflow counter on failure; never spin or sleep on the audio thread. |
