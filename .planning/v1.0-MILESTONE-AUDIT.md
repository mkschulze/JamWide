## Cross-Phase Integration Findings

### 1. Session Recording Not Wired in JUCE Plugin (REC-01, REC-02)

**Severity:** Tech Debt (not a blocker — implemented in Phase 1 CLAP/ImGui path)
**Finding:** No recording UI toggle, no `SetRecordingCommand`, no `config_savelocalaudio` write exists in the JUCE plugin codebase. Recording was implemented in Phase 1 for the legacy ImGui UI (`src/ui/ui_local.cpp:178-186`) and verified there.
**Impact:** Users of the JUCE plugin (VST3/AU/Standalone) cannot enable session recording. The feature exists only in the CLAP plugin.
**Recommendation:** Add recording toggle to JUCE UI in next milestone, or explicitly defer.

### 2. Codec Selector UI Mismatch (CODEC-03, CODEC-05)

**Severity:** Bug (visual)
**Finding:** `ConnectionBar.cpp:90` sets `codecSelector.setSelectedId(2)` (Vorbis), but `JamWideJuceProcessor.cpp:46` defaults NJClient to FLAC. The UI shows "Vorbis" while the actual encoding is FLAC.
**Impact:** User confusion — the codec selector doesn't reflect actual state on startup.
**Recommendation:** Change to `setSelectedId(1)` to match FLAC default, or read actual encoder format from processor.

### 3. Remote Codec Badge Always Empty (CODEC-03)

**Severity:** Tech Debt (cosmetic)
**Finding:** `ChannelStripArea.cpp:380-381` passes empty `codecStr` to `ChannelStrip.configure()`. `GetUserChannelCodec()` is available on NJClient but never called.
**Impact:** Users can't see what codec remote participants are using.
**Recommendation:** Populate `codecStr` from `GetUserChannelCodec()` during user info refresh.

### 4. Error Detail Dropped (UI-01, UI-03)

**Severity:** Minor
**Finding:** `StatusChangedEvent.error_msg` is drained and discarded in `JamWideJuceEditor::drainEvents()`. Status label uses hardcoded strings instead of server-provided error text.
**Impact:** Users see generic "Connection failed" instead of specific server error messages.
**Recommendation:** Surface `error_msg` in status label when available.
