Move codec call-sites that perform allocation/blocking I/O off the audio thread. **Call-site integration only — no codec algorithm or format edits.** Closes AUDIT CR-08 (`start_decode` on audio thread, sessionmode), **H-04** (`fread` in `DecodeState::runDecode`, IN STEADY STATE), and CR-11 partial (decoder internal allocations).

**Codex HIGH-1 review response (FOUNDATIONAL REDESIGN — incorporated):**

The original plan claimed to "structurally close" H-04 by moving `start_decode` and the initial fill loop off-thread. **Codex correctly identified that this is insufficient**: once the initial buffered samples are consumed, `mixInChannel` continues calling `chan->runDecode(256)`, which inside DecodeState::runDecode at njclient.cpp:256 reaches `fread(srcbuf, 1, sz, decode_fp)` for any DecodeState whose `decode_fp` is non-null. So H-04 is reachable IN STEADY STATE for sessionmode files, not just at startup. The original plan's "H-04 cleared" claim was a false-green.

This revision redesigns to Codex's option (b) — **file-reader thread / run-thread refill queue feeding the audio thread**:

1. **Construction phase (was already in original plan):** mixInChannel emits `DecodeArmRequest` onto SPSC. Run thread's drainArmRequests calls start_decode off-thread.

2. **NEW — split DecodeState into "audio-side" and "run-side" twins:** Run thread's start_decode produces TWO things:
   - A `DecodeState` for the audio thread that has `decode_fp = nullptr` and `decode_buf` set to a fresh `DecodeMediaBuffer*` (SPSC-backed from 15.1-07c).
   - A run-thread-side bookkeeping record (the `SessionmodeFileReader` struct) that holds the FILE*, a back-pointer to that DecodeMediaBuffer, and the read-position state.

   The audio-thread DecodeState is published via the existing PeerNextDsUpdate from 15.1-07a; the run-thread bookkeeping record goes onto a new `m_sessionmode_file_readers` list (run-thread-private, doesn't cross thread boundaries).

3. **NEW — run-thread refill tick:** On every run-thread tick (alongside drainArmRequests / drainDeferredDelete), the run thread iterates `m_sessionmode_file_readers`, calls `fread()` on each active FILE*, and pushes the read bytes into the corresponding DecodeMediaBuffer via `DecodeMediaBuffer::Write` (lock-free SPSC push). The audio thread's `chan->runDecode(256)` calls reach `decode_buf->Read` (DecodeMediaBuffer SPSC drain — lock-free, addressed by 15.1-07c CR-12 fix). **It NEVER reaches the `fread` path because `decode_fp == nullptr` for every audio-thread-visible DecodeState.**

4. **Cleanup:** When a sessionmode DecodeState is freed (via the audio-thread-emitted deferred-delete in 15.1-05), the matching SessionmodeFileReader entry is identified by the DecodeMediaBuffer pointer and removed from the list; its FILE* is fclose'd on the run thread.

**Why this is genuine H-04 closure (not false-green):**
- The grep `fread.*decode_fp` still finds the fread (we don't delete `runDecode`), but the ONLY callers reachable from `AudioProc` invoke runDecode against a DecodeState whose `decode_fp == nullptr`. The if-branch `if (decode_fp)` at the top of runDecode is never taken on the audio thread.
- The auditor's re-run grep proves this structurally: every audio-thread-visible DecodeState construction site (the run-thread drainArmRequests body) sets `ds->decode_fp = nullptr` immediately after start_decode returns, transferring the actual FILE* to the SessionmodeFileReader bookkeeping record.

Output: CR-08 cleared; **H-04 cleared in steady state, not just at startup**; mixInChannel allocation-free and lock-free.

    ```cpp
    void NJClient::refillSessionmodeBuffers()
    {
        // Iterate m_sessionmode_file_readers; for each entry:
        //   - if buffer's refcnt indicates the audio thread has Released its share AND we have
        //     too, the entry is dead — fclose + remove.
        //   - else, top up the buffer with bytes from the FILE* if there's space.
        //
        // The lock-free SpscRing inside DecodeMediaBuffer has finite capacity (32 chunks per
        // 15.1-07c). We push at most a few chunks per tick to avoid stalling on a single
        // sessionmode file when many are active.
        constexpr int kMaxChunksPerTickPerFile = 4;

        for (size_t i = 0; i < m_sessionmode_file_readers.size(); /* manual advance */) {
            SessionmodeFileReader& rdr = m_sessionmode_file_readers[i];

            // Detect dead entry: if rdr.buffer's refcnt has dropped (i.e., the ds that held
            // the audio-thread reference was deferred-deleted and the run thread has drained
            // m_deferred_delete_q, which calls ~DecodeState which calls decode_buf->Release).
            // refcnt is 1 means only THIS reader holds a ref — the audio side is gone.
            //
            // We need a way to peek refcnt without modifying it. Add a public accessor on
            // DecodeMediaBuffer (15.1-07c may already provide one; if not, add it here):
            //     int GetRefCount() const noexcept { return m_refcnt.load(std::memory_order_relaxed); }
            if (rdr.buffer && rdr.buffer->GetRefCount() <= 1) {
                if (rdr.file) std::fclose(rdr.file);
                if (rdr.buffer) rdr.buffer->Release();  // refcnt → 0, deletes
                m_sessionmode_file_readers.erase(m_sessionmode_file_readers.begin() + i);
                continue;
            }

            // Active entry — top up the buffer.
            if (!rdr.eof && rdr.file) {
                for (int chunk = 0; chunk < kMaxChunksPerTickPerFile; ++chunk) {
                    uint8_t tmp[jamwide::CHUNK_BYTES];
                    size_t got = std::fread(tmp, 1, sizeof(tmp), rdr.file);
                    if (got == 0) {
                        rdr.eof = true;
                        break;
                    }
                    int written = rdr.buffer->Write(tmp, static_cast<int>(got));
                    if (written < static_cast<int>(got)) {
                        // Buffer is full — backed up. We've already read 'got' bytes from the

                        // audio thread is consuming continuously, so the stall is transient.
                        // Choose (a) for simplicity; document.
                        break;
                    }
                }
            }
            ++i;
        }
    }
    ```

    ```cpp
        TEST("HIGH-1: refill loop preserves byte-stream integrity end-to-end");
        // Stub: an in-memory "file" of 16384 bytes (4 chunks).
        constexpr int kFileSize = 16384;
        std::vector<uint8_t> stub_file(kFileSize);

        // Stub DecodeMediaBuffer using the SPSC primitives.
        jamwide::SpscRing<jamwide::DecodeChunk, 32> chunks;
        std::array<uint8_t, jamwide::CHUNK_BYTES * 2> consumer_buf{};
        int cb_len = 0, cb_pos = 0;

        auto buffer_write = [&](const uint8_t* data, int len) -> int {
            int remaining = len;
            const uint8_t* p = data;
            while (remaining > 0) {
                jamwide::DecodeChunk c{};
                c.len = std::min(remaining, jamwide::CHUNK_BYTES);
                std::memcpy(c.data, p, c.len);
                if (!chunks.try_push(c)) return len - remaining;
                p += c.len; remaining -= c.len;
            }
            return len;
        };
        auto buffer_read = [&](void* buf, int len) -> int {
            int written = 0; uint8_t* out = static_cast<uint8_t*>(buf);
            while (written < len) {
                int avail = cb_len - cb_pos;
                if (avail > 0) {
                    int take = std::min(avail, len - written);
                    std::memcpy(out + written, consumer_buf.data() + cb_pos, take);
                    cb_pos += take; written += take; continue;
                }
                auto c = chunks.try_pop();
                if (!c) break;
                std::memcpy(consumer_buf.data(), c->data, c->len);
                cb_len = c->len; cb_pos = 0;
            }
            return written;
        };

        // Simulate audio-thread runDecode → decode_buf->Read in parallel.
        std::vector<uint8_t> audio_received;
        size_t file_pos = 0;
        bool eof = false;
        while (!eof || !chunks.empty() || cb_len > cb_pos) {
            // Run-thread side refill (4 chunks per tick).
            for (int chunk = 0; chunk < 4 && !eof; ++chunk) {
                if (file_pos >= stub_file.size()) { eof = true; break; }
                size_t to_read = std::min<size_t>(jamwide::CHUNK_BYTES, stub_file.size() - file_pos);
                int written = buffer_write(stub_file.data() + file_pos, static_cast<int>(to_read));
                if (written < static_cast<int>(to_read)) {

                    break;  // buffer full

            }

            for (int call = 0; call < 8; ++call) {

                int got = buffer_read(tmp, sizeof(tmp));
                if (got <= 0) break;
                audio_received.insert(audio_received.end(), tmp, tmp + got);
            }
        }

        if (audio_received.size() == stub_file.size()
            && std::memcmp(audio_received.data(), stub_file.data(), stub_file.size()) == 0) {
            PASS();
        } else {
            FAIL("refill loop produced byte-stream divergence");
        }
    }
    ```
