TouchOSC .tosc format (from tosclib docs):
- File is zlib-compressed XML (no zlib header, raw deflate, or gzip — use zlib.compress())
- Root element: <lexml version="10" ...>
- Contains nested <node> elements with type attribute
- Node types: PAGE, GROUP, FADERV, FADERH, BUTTON, LABEL, ENCODER, TEXT, RADIALH, RADAR
- Properties stored as child elements: <property type="s"><key>name</key><value>...</value></property>
- OSC message config: <osc><message><triggers>x</triggers><path>/addr</path></message></osc>
- Layout: x, y, w, h properties (frame coordinates)
- Colors: <color type="c"><r>R</r><g>G</g><b>B</b><a>A</a></color>

**Template layout specifications (per D-12 through D-16):**
- Target: iPad landscape, 1024x768 base resolution
- Single page layout

**Layout zones:**

1. **Session Info Panel** (top-left, ~300x120):
   - Label "JamWide" as title
   - BPM display: LABEL bound to `/JamWide/session/bpm` (receive only)
   - BPI display: LABEL bound to `/JamWide/session/bpi` (receive only)
   - Beat display: LABEL bound to `/JamWide/session/beat` (receive only)
   - Status display: LABEL bound to `/JamWide/session/status` (receive only)
   - Users display: LABEL bound to `/JamWide/session/users` (receive only)
   - Connect BUTTON: sends string to `/JamWide/session/connect` with value `"ninbot.com:2049"` (default, user can edit in TouchOSC)
   - Disconnect BUTTON: sends float 1.0 to `/JamWide/session/disconnect`

2. **Master Section** (top-right of session panel, ~100x400):
   - FADERV bound to `/JamWide/master/volume` (range 0-1)
   - BUTTON for mute bound to `/JamWide/master/mute`
   - VU meter bars: 2 FADERV (read-only, no touch) bound to `/JamWide/master/vu/left` and `/vu/right`
   - LABEL "Master"

3. **Metronome Section** (next to master, ~100x400):
   - FADERV bound to `/JamWide/metro/volume` (range 0-1)
   - ENCODER/knob bound to `/JamWide/metro/pan` (range 0-1)
   - BUTTON for mute bound to `/JamWide/metro/mute`
   - LABEL "Metro"

4. **Local Channels 1-4** (left side, below session info, ~400x500):
   - 4 channel strips, each containing:
     - LABEL for channel name "Local {n}"
     - FADERV bound to `/JamWide/local/{n}/volume` (range 0-1)
     - ENCODER/knob bound to `/JamWide/local/{n}/pan` (range 0-1)
     - BUTTON for mute bound to `/JamWide/local/{n}/mute`
     - BUTTON for solo bound to `/JamWide/local/{n}/solo`
     - VU bars: 2 narrow FADERV (receive-only) bound to `/JamWide/local/{n}/vu/left` and `/vu/right`

5. **Remote Users 1-8** (right side, ~600x500, per D-13):
   - 8 remote user strips (2 rows of 4, or single row of 8 depending on space), each containing:
     - LABEL bound to `/JamWide/remote/{idx}/name` (receive only — displays username)
     - FADERV bound to `/JamWide/remote/{idx}/volume` (range 0-1)
     - ENCODER/knob bound to `/JamWide/remote/{idx}/pan` (range 0-1)
     - BUTTON for mute bound to `/JamWide/remote/{idx}/mute`
     - BUTTON for solo bound to `/JamWide/remote/{idx}/solo`

**TouchOSC XML structure:**

The script must produce valid TouchOSC XML. Key structure:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<lexml version="10">
  <node type="PAGE" name="JamWide">
    <!-- Session info group -->
    <node type="GROUP" name="session" x="0" y="0" w="300" h="120">
      <node type="LABEL" name="title">
        <property type="s"><key>text</key><value>JamWide</value></property>
        <!-- frame: x, y, w, h -->
      </node>
      <!-- ... more labels for BPM, BPI, etc. -->
    </node>
    <!-- Master group -->
    <node type="GROUP" name="master">
      <node type="FADERV" name="masterVol">
        <osc><message><triggers>x</triggers><path>/JamWide/master/volume</path></message></osc>
        <!-- properties for range, position, color -->
      </node>
      <!-- ... -->
    </node>
    <!-- ... Local channels, Remote users -->
  </node>
</lexml>
```

**IMPORTANT:** Study the actual .tosc XML format carefully. The exact element names, property encoding, and OSC message binding format MUST match what TouchOSC expects. Key details:
- Node properties are encoded as `<property>` children with `type` attribute (s=string, f=float, i=int, b=bool, c=color, r=rect)
- Frame/position uses a `<property type="r">` with `<x>`, `<y>`, `<w>`, `<h>` children
- OSC binding uses `<messages>` element inside the node, with `<osc>` sub-elements
- The final file is the XML compressed with Python's `zlib.compress()` at default level
