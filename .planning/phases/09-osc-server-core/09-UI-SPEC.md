# Phase 9 -- UI Design Contract

> Visual and interaction contract for the OSC Server Core phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native JUCE C++ plugin -- no web framework) |
| Preset | not applicable |
| Component library | JUCE LookAndFeel_V4 via `JamWideLookAndFeel` subclass |
| Icon library | none (custom painted shapes, inline SVG via `juce::Drawable`) |
| Font | JUCE default sans-serif (system font) |

**Note:** This is a JUCE audio plugin, not a web application. All UI is rendered via custom `paint()` methods and JUCE component classes. The design system is `JamWideLookAndFeel` defined in `juce/ui/JamWideLookAndFeel.h`. No shadcn, React, or CSS frameworks apply.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between status dot and label |
| sm | 8px | Compact element spacing, ConnectionBar horizontal padding, dialog field margins |
| md | 16px | Default element spacing, dialog section gaps |
| lg | 24px | Dialog outer padding (top/bottom/left/right) |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- OSC status dot diameter: 10px (circle, not rectangular -- does not follow 8-point grid)
- OSC status dot click target: 20x20px minimum (accessibility -- entire ConnectionBar right-side region)
- Dialog size constraint: ~200x300px per D-10 (width x height)

## Component Inventory

### Component 1: OscStatusDot

**Location:** Right side of `ConnectionBar`, after the existing route/sync/codec controls. Visually lives in the 44px-tall connection bar.

**Visual specification:**
- Filled circle, 10px diameter, centered vertically in ConnectionBar
- 4px gap between the dot and the "OSC" label text to its left
- Label: "OSC" in 9px regular, `kTextSecondary` color
- Total clickable area: minimum 44x44px (full ConnectionBar height, extends left to cover label)

**States:**

| State | Dot Color | Tooltip |
|-------|-----------|---------|
| Disabled (OSC off) | `kTextSecondary` (#8888AA) | "OSC: Off" |
| Active (listening + sending) | `kAccentConnect` (#40E070) | "OSC: Active (port 9000)" |
| Error (port bind failed) | `kAccentDestructive` (#E04040) | "OSC: Error" |

**Interaction:**
- Click anywhere on the dot or "OSC" label: opens OscConfigDialog via `CallOutBox::launchAsynchronously()`
- CallOutBox arrow points to the status dot's screen position
- No hover state change on the dot itself (the ConnectionBar already handles hover via LookAndFeel)

**Source:** D-08 (IEM-style footer status dot), D-09 (3-state indicator), OSC-09 requirement

### Component 2: OscConfigDialog

**Location:** CallOutBox popup anchored to OscStatusDot. Appears as a floating panel attached to the dot.

**Size:** 200px wide x 300px tall (per D-10). CallOutBox handles positioning and arrow.

**Background:** `kBgPrimary` (#1A1D2E) with 1px `kBorderSubtle` (#3A3D58) outline (inherited from CallOutBox default styling with LookAndFeel overrides).

**Layout (top to bottom, 24px outer padding):**

```
+------------------------------------------+
|  OSC Configuration            [title 13B] |
|  ---------------------------------------- |
|  [x] Enable OSC              [toggle 11R] |
|                                           |
|  Receive Port                 [label 11R] |
|  [ 9000                    ] [input 13R]  |
|                                           |
|  Send IP                      [label 11R] |
|  [ 127.0.0.1               ] [input 13R]  |
|                                           |
|  Send Port                    [label 11R] |
|  [ 9001                    ] [input 13R]  |
|                                           |
|  Feedback: 100ms (fixed)      [info  9R]  |
|                                           |
|  (error message area)         [err  11R]  |
+------------------------------------------+

B = bold, R = regular
```

**Field specifications:**

| Field | Component | Default Value | Validation |
|-------|-----------|---------------|------------|
| Enable OSC | `juce::ToggleButton` | Off (unchecked) | None |
| Receive Port | `juce::TextEditor` (numeric) | "9000" | 1-65535, integer only |
| Send IP | `juce::TextEditor` | "127.0.0.1" | Non-empty string (basic format check) |
| Send Port | `juce::TextEditor` (numeric) | "9001" | 1-65535, integer only |
| Feedback interval | `juce::Label` (read-only) | "100ms (fixed)" | Not editable -- display only |

**Field styling:**
- Text inputs: `kSurfaceInput` (#252840) background, `kTextPrimary` (#E0E0E0) text, `kBorderSubtle` (#3A3D58) outline, `kBorderFocus` (#5A6080) on focus
- Input height: 24px
- Label-to-input vertical gap: 4px
- Inter-field vertical gap: 8px
- Title separator: 1px `kBorderSubtle` line, 8px below title, 8px above first field

**Interaction:**
- Changes apply immediately on focus-loss or Enter key (no Apply/Cancel buttons -- matches IEM pattern)
- When Enable toggle is turned ON: attempt to bind receiver port and connect sender. If bind fails, show error and revert toggle to OFF
- When Enable toggle is turned OFF: stop receiver and sender, status dot goes grey
- Port/IP changes while enabled: stop, reconfigure, restart. If new port fails to bind, show error
- Tab order: Enable toggle -> Receive Port -> Send IP -> Send Port
- Escape or click outside: CallOutBox dismisses (changes already applied)

**Disabled state:** When Enable toggle is OFF, the port/IP fields remain editable (per D-11: "Users may want to disable OSC without clearing port settings"). Fields are NOT greyed out.

**Source:** D-08, D-10, D-11, D-17, D-18, OSC-03 requirement

### Component 3: Error Display (within OscConfigDialog)

**Location:** Bottom of OscConfigDialog, below the "Feedback" info line.

**Visual specification:**
- Text label, 11px regular
- Color: `kAccentDestructive` (#E04040)
- Visible only when an error exists (port bind failure)
- Wraps to 2 lines maximum within the 200px dialog width minus 48px padding (= ~152px text width)

**Source:** D-18 (port bind failure error display), Claude's discretion (exact wording)

## State Transitions

### OSC Status Dot State Machine

```
                     +----------+
                     | DISABLED |  (grey dot, initial state)
                     +-----+----+
                           |
                    Enable toggle ON
                           |
                     +-----v----+
              +----->|  BINDING |  (transient, < 100ms)
              |      +-----+----+
              |            |
              |    +-------+-------+
              |    |               |
              |  bind OK      bind FAIL
              |    |               |
              | +--v---+     +----v---+
              | |ACTIVE|     | ERROR  |  (red dot)
              | +--+---+     +----+---+
              |    |               |
              | disable         change port
              |    |               |
              |    +-------+-------+
              |            |
              +---< DISABLED >
```

### Dialog Field Interaction Flow

1. User clicks status dot -> CallOutBox opens with current config values
2. User toggles Enable ON -> immediate bind attempt -> dot turns green or red
3. User edits port while enabled -> on focus-loss: stop, rebind, dot updates
4. User toggles Enable OFF -> stop receiver/sender -> dot turns grey, fields stay
5. User clicks outside dialog -> CallOutBox dismisses -> settings already persisted

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Keyboard navigation | Tab into dialog fields. Enter to apply. Escape to dismiss. |
| Screen reader | ToggleButton provides built-in JUCE accessibility ("Enable OSC, checkbox, checked/unchecked") |
| Color-only indicators | Status dot color is supplemented by tooltip text ("OSC: Off / Active / Error") |
| Touch targets | Status dot click region: full ConnectionBar height (44px) x 44px wide minimum |
| Focus indicators | `kBorderFocus` (#5A6080) outline on focused text fields (already in LookAndFeel) |
