# Phase 14 — UI Design Contract

> Visual and interaction contract for MIDI Remote Control UI elements. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native C++ JUCE plugin) |
| Preset | not applicable |
| Component library | Custom JUCE LookAndFeel_V4 subclass (JamWideLookAndFeel) |
| Icon library | None (all indicators are custom-painted primitives: dots, rectangles, borders) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap (e.g., "MIDI" label to dot), inline padding in table rows |
| sm | 8px | Compact element spacing (dialog field margins, popup menu item padding) |
| md | 16px | Default element spacing (dialog section gaps, overlay padding) |
| lg | 24px | Section padding (dialog top/bottom padding, between mapping table and buttons) |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- MidiStatusDot total width: 44px (matching OscStatusDot placement in ConnectionBar footer)
- Status dot diameter: 8px (matching OSC dot)
- MIDI Learn overlay minimum touch target: 44px height (matching VbFader kThumbDiameter)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 (regular) | 1.4 |
| Label | 9px | 400 (regular) | 1.0 |
| Heading | 14px | 600 (semibold) | 1.2 |

Notes:
- Label (9px): Used for "MIDI" text next to status dot and mapping table column headers. Matches "OSC" label in OscStatusDot (9.0f FontOptions).
- Body (13px): Used for dialog field labels, mapping table cell text, popup menu items, and MIDI Learn overlay text ("Waiting for CC...", "CC 7 Ch 1").
- Heading (14px semibold): Used for dialog title ("MIDI Configuration") and section dividers within the config dialog.

## Component Inventory

### 1. MidiStatusDot (footer indicator)

**Location:** ConnectionBar, positioned 44px to the left of the OscStatusDot (or leftward in the right-aligned control group).

**Visual specification:**
- Total bounds: 44px wide x full footer height (matching OscStatusDot)
- Layout: "MIDI" label (9px, kTextSecondary) + 4px gap + 8px diameter circle, centered vertically
- 3-state color:
  - Grey (kTextSecondary #8888AA): No mappings configured / MIDI off
  - Green (kAccentConnect #40E070): Active -- receiving MIDI CC messages
  - Red (kAccentDestructive #E04040): Error state (device open failure in standalone)

**Interaction:**
- Click: Opens MidiConfigDialog via juce::CallOutBox (same pattern as OscStatusDot -> OscConfigDialog)
- Hover: Tooltip shows state text ("MIDI: Off", "MIDI: Active (4 mappings)", "MIDI: Error")
- Timer poll: 500ms repaint interval (matching OscStatusDot)

### 2. MidiConfigDialog (CallOutBox popup)

**Location:** Launched from MidiStatusDot click, anchored to MidiStatusDot screen bounds.

**Visual specification:**
- Size: 320px wide x 400px tall (wider than OscConfigDialog's 200x300 to accommodate mapping table)
- Background: kBgElevated (#222540)
- Border radius: 0px (JUCE CallOutBox handles arrow/border)
- Internal padding: 16px all sides

**Layout (top to bottom):**
1. **Section: Standalone Device Selector** (only visible when `JucePlugin_Build_Standalone == 1`)
   - Heading: "MIDI Devices" (14px semibold, kTextHeading)
   - Input device ComboBox: full width, 24px height
   - Output device ComboBox: full width, 24px height
   - 8px gap between ComboBoxes
   - 16px gap after section

2. **Section: Mapping Table**
   - Heading: "Mappings" (14px semibold, kTextHeading) + mapping count badge "(4)"
   - Column headers (9px, kTextSecondary): Parameter | CC# | Ch | (delete)
   - Table rows: 24px height, alternating kSurfaceStrip / kBgPrimary
   - Each row: parameter name (13px, kTextPrimary) | CC number (13px) | Channel (13px) | X button (kAccentDestructive on hover)
   - Scrollable if > 8 rows visible (192px max table height)
   - Empty state: centered "No MIDI mappings" (13px, kTextSecondary)

3. **Section: Actions**
   - "Clear All" button: kAccentDestructive text, no fill, right-aligned
   - 16px top padding from table

### 3. MIDI Learn Overlay (transient feedback)

**Trigger:** User right-clicks a fader/knob/button, selects "MIDI Learn" from popup menu.

**Visual specification (Waiting state):**
- Target component gets a 2px border in kAccentWarning (#CCB833)
- Border pulses (opacity oscillates 0.6 to 1.0 over 600ms cycle)
- Small overlay label appears 4px above or below the component:
  - Background: kSurfaceOverlay (#1E2138) with 1px kBorderSubtle border
  - Text: "Waiting for CC..." (13px, kAccentWarning)
  - Padding: 8px horizontal, 4px vertical
  - Corner radius: 4px

**Visual specification (Learned state):**
- Border changes to kAccentConnect (#40E070), solid (no pulse)
- Overlay text changes to "CC {N} Ch {M}" (13px, kAccentConnect)
- Auto-dismisses after 1200ms
- On dismiss: border fades out over 300ms

**Visual specification (Cancel):**
- If user right-clicks again or presses Escape during "Waiting": overlay disappears immediately, border removed
- No confirmation needed for cancel

### 4. Right-Click Context Menu (PopupMenu)

**Trigger:** Right-click on any VbFader, pan slider, mute button, or solo button.

**Menu items:**
- "MIDI Learn" -- always shown
- "Clear MIDI" -- shown only when parameter already has a mapping (greyed out otherwise)
- Separator line
- (No scale menu items -- scale menu moves to empty-space right-click only)

**Visual specification:**
- Uses JamWideLookAndFeel default popup menu rendering
- Background: kBgElevated
- Text: kTextPrimary (13px)
- Highlighted item: kSurfaceStrip background
- Disabled item: kTextSecondary

## Interaction Contracts

### MIDI Learn Flow

1. User right-clicks a fader/knob/button
2. PopupMenu appears with "MIDI Learn" (+ "Clear MIDI" if mapped)
3. User clicks "MIDI Learn"
4. Target component shows pulsing kAccentWarning border + "Waiting for CC..." overlay
5. User moves a CC on their MIDI controller
6. Overlay changes to "CC {N} Ch {M}" with green border
7. After 1200ms, overlay auto-dismisses
8. Mapping is immediately active (no save button needed)

### MIDI Learn Timeout

- After 10 seconds with no CC received, the Learn state auto-cancels
- Overlay disappears, border removed, no error message shown
- User can retry by right-clicking again

### MIDI Learn Conflict Resolution

- If the CC+Channel combination is already mapped to another parameter:
  - Silently reassign to the new parameter (last-write-wins)
  - Old parameter loses its mapping without notification

### Status Dot State Transitions

| Condition | Dot State | Tooltip |
|-----------|-----------|---------|
| No mappings configured | Grey | "MIDI: Off" |
| Mappings exist, CC received in last 2 seconds | Green | "MIDI: Active ({N} mappings)" |
| Mappings exist, no recent CC activity | Green (steady, not pulsing) | "MIDI: Active ({N} mappings)" |
| Standalone device open failed | Red | "MIDI: Error" |

### Clear All Confirmation

- "Clear All" button in MidiConfigDialog shows a juce::AlertWindow confirmation
- Title: "Clear All MIDI Mappings"
- Body: "Remove all MIDI mappings?"
- Buttons: "Clear All" (kAccentDestructive) | "Cancel"
