## Deviations from Plan

### Auto-fixed during execution

**1. [Plan-faithful expansion] Test link dependencies — pure-C++ pattern over codec-link prescription**
- **Found during:** Task 2 — registering `test_peer_churn_simulation` in CMakeLists.txt.
- **Issue:** The executor prompt's deviation-rule guidance prescribed link dependencies "matches the pattern from 15.1-07c and 15.1-08" (which would suggest `wdl + vorbis + vorbisenc + ogg + FLAC`). However, 15.1-07c (test_decode_media_buffer_spsc) is actually pure-C++ with NO link deps, matching the pattern of every OTHER SPSC test (`test_remote_user_mirror` / `test_local_channel_mirror` / `test_decode_state_arming` / `test_block_queue_spsc`). Only `test_decoder_prealloc` (15.1-08) has codec link deps because it directly constructs `VorbisDecoder` / `FlacDecoder` instances. `test_peer_churn_simulation` doesn't construct decoders — it uses stand-in `jamwide::DecodeState` POD with a marker field.
- **Fix:** Followed the existing pure-C++ no-link pattern (matches `test_decode_media_buffer_spsc` from 15.1-07c). The test compiles and runs cleanly under both Release and TSan.
- **Files modified:** `CMakeLists.txt` (no codec link added).
- **Verification:** Build green, test PASSES under both Release and TSan with zero TSan reports.
- **Committed in:** `4551a4f` (Task 2 commit).

**2. [Plan-faithful expansion] Three-thread test design over plan's two-thread skeleton**
- **Found during:** Task 2 — designing the test harness based on the plan's skeleton.
- **Issue:** The plan's skeleton showed two threads (audio + run); but the plan's own action-step pattern has the run thread emitting `DecodeArmRequest` AND draining the result, AND publishing `PeerNextDsUpdate` into the publish ring. The reaper thread is needed because, without it draining `dd_q` concurrently, the run thread's `drain_arms` stalls once `arm_q` saturates because `dd_q` (used as the orphan-defer fallback when ru_q is full) has nowhere to publish to. Same architectural fix that 15.1-09 Task 3 Deviation #3 already documented in `tests/test_decode_state_arming.cpp`.
- **Fix:** Added a third reaper thread that drains `dd_q` at 1ms cadence; coordinated via `stop` atomic.
- **Files modified:** `tests/test_peer_churn_simulation.cpp`.
- **Verification:** Test PASSES under both Release (0.08s) and TSan (0.31s); no deadlocks observed across multiple runs.
- **Committed in:** `4551a4f` (Task 2 commit).

**3. [Plan-faithful documentation] Architecture-mirror finding line number shifted from cited 1405 to actual 1648-1653**
- **Found during:** Task 3 — locating the architecture-mirror dead-but-live finding for inclusion in 15.1-AUDIT-final.md.
- **Issue:** 15.1-MIRROR-AUDIT.md cited the auth-reply lobby renormalize at `njclient.cpp:1405`; the actual current location after subsequent edits is lines 1648-1653 (the same code structurally, in the same MESSAGE_SERVER_AUTH_REPLY handler, just shifted by edits between 15.1-07a landing and now). - **Fix:** Used the current line number (1648-1653) in 15.1-AUDIT-final.md and deferred-items.md while explicitly noting the citation shift; the architectural finding is unchanged.
- **Files modified:** `15.1-AUDIT-final.md`, `deferred-items.md`.
- **Committed in:** `7ee8c47` (Task 3 commit).
