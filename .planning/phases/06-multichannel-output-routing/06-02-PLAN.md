<objective>
Wire the Route button UI in ConnectionBar and connect routing selectors on channel strips to the command infrastructure built in Plan 01. Users can switch routing modes and override individual strip routing.

Purpose: This delivers MOUT-02 (auto-assign by user) and MOUT-03 (auto-assign by channel) through the quick-assign button, and enables MOUT-01 per-strip routing control. Without this, the audio routing backend has no user-facing controls.

Output: Route button in ConnectionBar with popup menu, updated routing selector items on channel strips (15 remote buses, metronome-reserved bus excluded), callbacks wired through command queue, selectors refresh from snapshot.

- MEDIUM: files_modified now includes ChannelStrip.h, ChannelStrip.cpp, JamWideJuceEditor.cpp

- LOW: Routing selector shows 15 items (Remote 1-15), not 16, since Remote 16 is reserved for metronome
- LOW: Menu item ID-to-mode mapping uses named constants for clarity
