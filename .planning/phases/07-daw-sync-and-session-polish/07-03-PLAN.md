**A. Create `.planning/references/OSC-EVALUATION.md`:**

Per D-19: Half-page, quick feasibility verdict.

**Document structure:**
1. **Summary** -- One sentence: OSC is widely supported but inconsistently implemented across DAWs for tempo/transport sync.
2. **Per-DAW Support Matrix** -- Table with columns: DAW, OSC Support, Transport Control, Tempo Sync, Notes

| DAW | OSC Support | Transport Control | Tempo Sync | Notes |
|-----|-------------|-------------------|------------|-------|
| REAPER | Native (built-in) | Yes (play/stop/record) | Yes (BPM read/write) | Best OSC support of any DAW |
| Logic Pro | Via Environment/IAC | Limited | No native BPM OSC | Requires workaround via MIDI |
| Ableton Live | Via Max for Live | Yes (M4L required) | Yes (M4L) | Requires Max license |
| Bitwig Studio | Native controller API | Yes | Yes | Good support, custom controller scripts |
| Pro Tools | None native | No | No | No OSC support |
| FL Studio | None native | No | No | No OSC support |
| Cubase/Nuendo | Via Generic Remote | Limited | Limited | Inconsistent |
| Studio One | None native | No | No | No OSC support |

3. **Feasibility Verdict** -- OSC is viable for REAPER and Bitwig. Cross-DAW coverage is poor (only ~30% of major DAWs have native support). Recommend as optional power-user feature, not primary sync mechanism. JamWide's AudioPlayHead approach (Phase 7) is the universal solution.
4. **Requirements Addressed** -- XSYNC-01, XSYNC-02 (from REQUIREMENTS.md v2)

**B. Create `.planning/references/MCP-ASSESSMENT.md`:**

Per D-19: Half-page, quick feasibility verdict.

**Document structure:**
1. **Summary** -- One sentence: MCP (Model Context Protocol) is designed for AI tool integration, not real-time DAW control -- wrong tool for the job.
2. **What MCP Is** -- Brief: Anthropic's protocol for giving AI models access to tools and data sources. JSON-RPC based, designed for request/response patterns with language models.
3. **Why It Doesn't Fit DAW Sync** -- MCP is request/response, not streaming. DAW transport sync requires sub-millisecond timing. MCP adds AI inference latency. No DAW implements MCP natively. The "bridge" concept (MCP server that controls a DAW) is theoretically possible but adds complexity without benefit over OSC/MIDI.
4. **Where MCP Could Help JamWide** -- Session management (AI-assisted server selection), chat (AI-powered translation, moderation), preset management. These are non-real-time tasks where MCP's strengths apply.
5. **Feasibility Verdict** -- Not feasible for DAW sync (XSYNC-03). Potentially valuable for non-real-time features in v3+.
6. **Requirements Addressed** -- XSYNC-03 (from REQUIREMENTS.md v2)
