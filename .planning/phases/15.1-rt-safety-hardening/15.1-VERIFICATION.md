## Signal 1b — Automated peer-churn simulation (closes ROADMAP-15.1-S1 automated portion)

Per Codex M-5: this signal is the **automated NINJAM-like peer-churn** previously referenced by
ROADMAP-15.1-S1 ("TSan build of test binaries passes a NINJAM session simulation with peer churn at
interval boundaries") but not actually implemented until this plan. The new test
`tests/test_peer_churn_simulation.cpp` ships in Task 2 of THIS plan and lands the automated coverage
that closes the gap.

**Test design:** three concurrent threads (audio + run + reaper) cycle 1000 NINJAM-shaped peer-churn
patterns through the production SPSC primitives + payloads from 15.1-04 (`spsc_payloads.h`). The
producer publishes `PeerAddedUpdate → PeerChannelMaskUpdate → PeerVolPanUpdate → DecodeArmRequest →
drainArmRequests-result-via-PeerNextDsUpdate → LocalChannelAddedUpdate →
LocalChannelMonitoringUpdate → PeerRemovedUpdate → LocalChannelRemovedUpdate` per cycle; the audio
thread drains both queues at the top of each tick and bumps `audio_drain_gen` (mirrors AudioProc); the
reaper drains the DecodeState* deferred-delete queue at 1ms cadence (mirrors NinjamRunThread::run).

The pure-C++ test deliberately mirrors the production audio-path data flow without linking NJClient,
so the SPSC primitives + payload contracts are exercised in isolation — same architectural coverage
as `test_remote_user_mirror` (15.1-07a), `test_local_channel_mirror` (15.1-06),
`test_decode_state_arming` (15.1-09).

**Acceptance gates:**
1. TSan run reports zero data races (the M-5 automated-coverage requirement)
2. All four drop counters (`ru_drops`, `lc_drops`, `arm_drops`, plus the implicit deferred-delete
   counter exercised via the reaper) are zero at end of test (Codex M-8 phase-close gate enforced
   in-test, not just in production)
3. `audio_drain_gen` made progress (proves the audio thread observed the run thread's publishes)

**Command (Release):** `cd build-test && ctest -R peer_churn_simulation --output-on-failure`
**Command (TSan):** `cd build-tsan && ctest -R peer_churn_simulation --output-on-failure`

**Date run:** 2026-04-28

**Result (Release, build-test/):** 1/1 PASSED, 0.08 sec wall.
**Result (TSan, build-tsan/):** 1/1 PASSED, 0.31 sec wall, **zero ThreadSanitizer reports**.

**Cycles executed:** 1000 peer-churn + arm + local-channel cycles per test run.
**Threads coordinated:** 3 (audio + run + reaper).

**Test file:** `tests/test_peer_churn_simulation.cpp` (~370 lines including test framework boilerplate
