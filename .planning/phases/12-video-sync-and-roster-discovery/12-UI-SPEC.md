#### Roster Strip States

| Condition | Roster Strip Visibility | Content |
|-----------|------------------------|---------|
| No roster received yet | Hidden | -- |
| Roster with 0 users | Hidden | -- |
| Roster with 1+ users | Visible | One pill per user |
| WebSocket disconnected | Remains visible with last known roster (VDO.Ninja continues independently) |

### 3. Buffer Delay Relay (Invisible)

No visible UI. The companion page receives `{"type":"bufferDelay","delayMs":N}` via WebSocket and forwards it to the VDO.Ninja iframe via `postMessage({setBufferDelay: N}, '*')`. The only visible effect is that VDO.Ninja's internal buffering changes, which the user perceives as video being "in sync" with the NINJAM interval.

### 4. Room Security Hash (Invisible)

No visible UI. The companion URL includes `#hash=XXXXXXXX` in the fragment when the NINJAM server has a password. This is constructed by the plugin before launching the browser.

## Interaction Contracts

### Bandwidth Dropdown Change Flow

```
User selects a new bandwidth profile from dropdown
  |
  +-- Save selection to localStorage (key: "jamwide-bandwidth-profile")
  |
  +-- IF VDO.Ninja iframe is loaded (config received):
  |     Rebuild VDO.Ninja URL with new &quality and &maxvideobitrate params
  |     Replace iframe src (triggers iframe reload)
  |     Cache most recent bufferDelay value
  |     Re-apply bufferDelay via postMessage after iframe load event
  |
  +-- IF no iframe loaded yet:
        Selection saved; will be used when iframe loads
```

### Roster Label Update Flow

```
WebSocket receives "roster" message
  |
  +-- IF roster.users.length === 0:
  |     Hide #roster-strip
  |
  +-- IF roster.users.length > 0:
        Clear existing pills in #roster-strip
        For each user in roster:
          Create pill with user.name as textContent
          Apply green left border (matched) -- all roster users are matched by convention
        Show #roster-strip
```

### Buffer Delay Relay Flow

```
WebSocket receives "bufferDelay" message
  |
  +-- Cache delayMs value in module-level variable
  |
  +-- IF VDO.Ninja iframe is loaded AND iframe load event has fired:
  |     Call iframe.contentWindow.postMessage({setBufferDelay: delayMs}, '*')
  |
  +-- IF iframe not yet loaded:
        Value is cached; will be applied when iframe fires load event
```

### Iframe Load Event Handler

```
VDO.Ninja iframe fires "load" event
  |
  +-- IF cached bufferDelay value exists:
        Call iframe.contentWindow.postMessage({setBufferDelay: cachedDelayMs}, '*')
```

Phase 12 adds two new state dimensions to the companion page:

| Roster State | Roster Strip | Pill Appearance |
|-------------|-------------|-----------------|
| No roster received | Hidden | -- |
| Empty roster (0 users) | Hidden | -- |

| Buffer Delay State | Cached Value | postMessage Sent |
