# Plan 12.1-01: Fix Video-Audio Sync Pipeline + Manual Delay Slider

**Objective:** Debug and fix the setBufferDelay pipeline so video is delayed to match NINJAM audio latency, and add a manual delay slider as a user-adjustable override.

**Spec:** `docs/superpowers/specs/2026-04-13-video-audio-sync-fix-design.md`

## Context

When users clap on camera, receivers see the clap instantly (~100-300ms via WebRTC) but hear audio ~8 seconds later (one NINJAM interval at 120 BPM / 16 BPI). The Phase 12 `setBufferDelay` mechanism exists but isn't working — video arrives undelayed. This plan fixes the pipeline and adds manual override.

## Interfaces

- `BufferDelayMessage { type: "bufferDelay", delayMs: number }` — existing WebSocket message from plugin
- `applyBufferDelay(delayMs: number)` — existing function in ui.ts that sends postMessage to iframe
- `buildVdoNinjaUrl()` — existing function that constructs the iframe URL with parameters
