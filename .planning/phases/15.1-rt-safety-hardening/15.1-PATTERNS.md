### `src/threading/spsc_payloads.h` (header, request-response)

**Analog:** `src/threading/ui_command.h`

This is the canonical project pattern: a header in `src/threading/` that defines pure-POD command structs and gathers them into a `std::variant<...>` typedef. The new payload header for 15.1 is a near-clone for typed records: `RemoteUserUpdate`, `LocalChannelUpdate`, `BlockRecord`, `DecodeChunk`, `DeferredDelete`.

**Header guard + namespace pattern** (`src/threading/ui_command.h:1-12`):
```cpp
/*

    Command types for UI thread → Run thread communication
*/

#define UI_COMMAND_H

#include <string>
#include <variant>

namespace jamwide {
```

**Variant-of-PODs pattern** (`src/threading/ui_command.h:104-120`):

**Per-record POD shape** (`src/threading/ui_command.h:34-44`):
```cpp
struct SetLocalChannelMonitoringCommand {
    int channel = 0;
    bool set_volume = false;
    float volume = 1.0f;
    bool set_pan = false;
    float pan = 0.0f;
    bool set_mute = false;
    bool mute = false;
    bool set_solo = false;
```

**What to copy verbatim:**
- File-banner format (project name + filename + role line)
- `#ifndef`/`#define`/`#endif` guard with all-caps filename
- `namespace jamwide { ... }` wrapping
- One `struct` per record, default-initialized scalar fields, no constructors, no methods
- The `using XxxUpdate = std::variant<...>;` typedef at the bottom

**What to add (new for 15.1, not present in `ui_command.h`):**
- `static_assert(std::is_trivially_copyable_v<RemoteUserUpdate>);` lock-in for each variant (verifies SPSC trivial-copy contract — see `15.1-RESEARCH.md` Use 2)
- `BlockRecord` carries an inline `float samples[MAX_BLOCK_SAMPLES * 2]` — distinguished from the UiCommand payloads which use `std::string` (heap-backed and NOT trivially copyable; `ui_command.h` is consumed by the run thread under a mutex, so std::string is fine there. The audio-thread queues in `spsc_payloads.h` MUST be trivial-copy.)

### `tests/test_spsc_state_updates.cpp` (test, event-driven — round-trip + concurrent)

**Analog:** `tests/test_encryption.cpp` framework, `src/threading/spsc_ring.h` API.

**SpscRing producer/consumer API to exercise** (`src/threading/spsc_ring.h:51-116`):

```cpp
bool try_push(const T& value);                  // returns false if full
std::optional<T> try_pop();                     // returns nullopt if empty
```

**Recommended test layout (skeleton, derived from `ui_command.h` consumer in `juce/NinjamRunThread.cpp:680-684`):**
```cpp
static void test_remote_user_update_roundtrip() {
    TEST("RemoteUserUpdate variant survives SPSC roundtrip");
    jamwide::SpscRing<jamwide::RemoteUserUpdate, 64> ring;
    jamwide::PeerAddedUpdate p{ /*slot=*/3 };
    bool pushed = ring.try_push(p);
    auto got = ring.try_pop();
    if (pushed && got && std::holds_alternative<jamwide::PeerAddedUpdate>(*got)
        && std::get<jamwide::PeerAddedUpdate>(*got).slot == 3)
        PASS();
    else
        FAIL("roundtrip failed");
}
```

**Concurrent-test skeleton:** spawn one producer `std::thread` calling `try_push` in a tight loop, one consumer `std::thread` calling `drain([](T&&){ ... })`. Run under TSan. Assert all pushed records observed exactly once. Loops × workload × `static_assert(std::is_trivially_copyable_v<T>)` for each variant.

### `tests/test_deferred_delete.cpp` (test, streaming)

**Analog:** `tests/test_encryption.cpp` framework, `src/threading/spsc_ring.h:51-116`.

**Test scenario (derived from RESEARCH § Use 1):** simulate audio-thread push of `DecodeState*` at 50 Hz × 256 deletes in a single burst (interval-boundary worst case), drain on a separate thread, verify (a) zero overflow, (b) each destructor runs exactly once on the consumer thread.

**Track destructor calls via test-double:**
```cpp
struct FakeDecodeState {
    int* dtor_count;
    ~FakeDecodeState() { ++(*dtor_count); }
};

static void test_deferred_delete_drains_completely() {
    TEST("256 deferred deletes all execute on consumer thread");
    int dtor_count = 0;
    jamwide::SpscRing<FakeDecodeState*, 256> ring;
    for (int i = 0; i < 256; ++i)
        assert(ring.try_push(new FakeDecodeState{&dtor_count}));
    ring.drain([](FakeDecodeState* p){ delete p; });
    if (dtor_count == 256) PASS(); else FAIL("destructor count mismatch");
}
```

## Shared Patterns

### Pattern A — Trivially-copyable variant for SPSC payloads

**Source:** `src/threading/ui_command.h:104-120` (existing) + new `static_assert` lock-in.

**Apply to:** `spsc_payloads.h`, every test file that touches a payload variant.

```cpp
using RemoteUserUpdate = std::variant<
    PeerAddedUpdate,
    PeerRemovedUpdate,
    PeerChannelMaskUpdate,
    PeerVolPanUpdate,
    PeerNextDsUpdate,
    PeerCodecSwapUpdate
>;

static_assert(std::is_trivially_copyable_v<RemoteUserUpdate>,
              "RemoteUserUpdate must be trivially copyable for SPSC handoff");
```

The `static_assert` is **new** for 15.1 — `ui_command.h` doesn't have it because `ConnectCommand` carries `std::string`s and is consumed under a JUCE `ScopedLock` (not on the audio thread). The new audio-thread-consumed payloads must lock the trivial-copy contract because the SPSC `try_push(const T&)`/`try_pop()` rely on `T` being either trivially copyable or movable (per `spsc_ring.h:29` doc comment), and trivially copyable is the safest choice for the audio thread.

### Pattern B — std::memory_order release/acquire pairing

**Source:** `juce/NinjamRunThread.cpp:661` (writer) and `juce/JamWideJuceProcessor.cpp:474` (reader).

**Apply to:** every new `std::atomic<...>` field added in 15.1-02 plus the synchronization flag for run-thread → audio-thread state-update queues.

```cpp
// Writer (run thread):
client->cached_status.store(currentStatus, std::memory_order_release);

// Reader (audio thread):
if (client && client->cached_status.load(std::memory_order_acquire) == NJC_STATUS_OK) ...
```

For `m_beatinfo_updated` / `m_bpm` / `m_bpi`: payload writes are `relaxed`, the synchronizing flag write is `release`; the matching read is `acquire` on the flag, then `relaxed` on payload. Per RESEARCH § Memory ordering analysis.

### Pattern C — drain-then-visit consumer

**Source:** `juce/NinjamRunThread.cpp:680-684` (`processCommands`).

**Apply to:** every consumer of an SPSC variant queue — both run-thread drains (deferred-delete, arm-requests) and audio-thread drains (RemoteUserUpdate, LocalChannelUpdate).

```cpp
processor.cmd_queue.drain([&](jamwide::UiCommand&& cmd) {
    const juce::ScopedLock sl(processor.getClientLock());   // run-thread only; audio-thread drain has NO lock
    std::visit([&](auto&& c) {
        using T = std::decay_t<decltype(c)>;

        if constexpr (std::is_same_v<T, jamwide::ConnectCommand>) {
            // ...
        }
        else if constexpr (std::is_same_v<T, jamwide::SetLocalChannelInfoCommand>) {
            // ...
        }
    }, cmd);
});
```

**Critical override for audio-thread drains:** the `juce::ScopedLock` line MUST be omitted on the audio-thread side — that's the entire point of the migration. Audio-thread drains run at the top of `AudioProc` with no synchronization beyond the SPSC ring's own atomics.
