# Phase 13 — UI Design Contract

> Visual and interaction contract for the Video Display Modes and OSC Integration phase. Three UI surfaces: companion popout page (new `popout.html`), companion main page (roster pill click handling), and TouchOSC template (video control section). No new JUCE plugin visual components -- OSC wiring is logic-only. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla CSS + JUCE custom LookAndFeel) |
| Preset | not applicable |
| Component library | none (companion page), JUCE LookAndFeel_V4 (plugin) |
| Icon library | Inline SVG (companion page) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` (companion page). JUCE default font (plugin side -- no new plugin UI in this phase). |

### Surface Breakdown

This phase modifies three surfaces. No new JUCE plugin visual components are introduced.

| Surface | Technology | Scope | Primary Focal Point |
|---------|-----------|-------|---------------------|
| Companion main page | HTML / CSS / TypeScript (Vite) | Roster pill click-to-popout, popout window tracking, deactivate message handling | Roster pill strip (clickable pills that open popout windows) |
| Companion popout page | HTML / CSS / TypeScript (Vite) | New `popout.html` -- solo VDO.Ninja iframe with name label + disconnect overlay | Solo video stream (fills 95%+ of viewport) |
| TouchOSC template | `.tosc` XML | New VIDEO section: active toggle + 8 popout trigger buttons | Video control panel in TouchOSC |

Extends the Phase 12 roster strip pills to be clickable for popout.

| Duration | Auto-dismiss after 5 seconds, or click to dismiss immediately |
| Placement | Absolute positioned at top of `#main-area`, full width, z-index 30 |

## Interaction Contracts

### Roster Pill Click-to-Popout Flow (Companion Main Page)

```
User clicks a roster pill
  |
  +-- Read data-stream-id attribute from the pill
  |
  +-- Check popout window Map for this stream ID
        |
        +-- IF window exists AND not closed:
        |     Call existingWindow.focus()
        |     Return (no new window)
        |
        +-- IF window does not exist OR is closed:
              Build popout URL: popout.html?room=...&push=...&view={streamId}&password=...&quality=...
              Call window.open(url, "jamwide-popout-{streamId}", features)
                |
                +-- IF window.open returns null (popup blocked):
                |     Show popup-blocked banner (auto-dismiss 5s)
                |     Return
                |
                +-- IF window opened successfully:
                      Store Window reference in Map<streamId, Window>
                      Add active-popout indicator (green left border) to the pill
```

### OSC Popout Trigger Flow (Plugin to Companion)

```
TouchOSC sends /JamWide/video/popout/{idx} 1.0
  |
  +-- OscServer::handleOscOnMessageThread() receives message
  |
  +-- Validate idx is within roster range (1-based to 0-based conversion)
        |
        +-- IF idx > roster size:
        |     Ignore silently (no error feedback, consistent with remote user pattern)
        |
        +-- IF valid:
              Look up roster[idx-1].streamId
              Call VideoCompanion::requestPopout(streamId)
                |
                +-- VideoCompanion sends WebSocket message:
                |     {"type":"popout","streamId":"..."}
                |
                +-- Companion ws-client.ts dispatches onPopout callback
                |
                +-- main.ts onPopout handler:
                      Same logic as roster pill click (check Map, open/focus window)
```

### OSC Video Active Toggle Flow

```
TouchOSC sends /JamWide/video/active 1.0
  |
  +-- OscServer::handleOscOnMessageThread() receives message
  |
  +-- IF value >= 0.5 (activate):
  |     Equivalent to user clicking Video button while inactive
  |     Plugin launches companion (same flow as button click, privacy modal skipped if already shown this session)
  |
  +-- IF value < 0.5 (deactivate):
        Plugin sends {"type":"deactivate"} to all WS clients
        Companion closes all popout windows via window.close()
        Companion shows connection-lost state
        Plugin stops WS server
        Plugin sets video button to inactive state
```

### Deactivate Flow (All Popouts Close)

```
Video deactivated (button toggle off OR OSC /video/active 0.0)
  |
  +-- Plugin VideoCompanion::deactivate():
        1. Send {"type":"deactivate"} WebSocket message to all connected clients
        2. Stop WebSocket server
        3. Set active_ = false
  |
  +-- Companion receives deactivate message:
        1. Iterate popout window Map
        2. Call window.close() on each open popout
        3. Clear the Map
        4. Remove all active-popout pill indicators
        5. Show connection-lost state in main page
```

### Popout Window Disconnect/Reconnect

```
Main companion receives roster update via WebSocket
  |
  +-- For each open popout in the Map:
        Send postMessage({type:'roster', users}) to the popout window
  |
  +-- Popout window receives postMessage:
        Check if own streamId is in the users array
          |
          +-- IF present:
          |     Hide disconnect overlay (if showing)
          |     Stream resumes automatically (VDO.Ninja handles reconnection)
          |
          +-- IF absent:
                Show disconnect overlay with username + "Disconnected"
```

## State Mapping

### Popout Window States

| Main WS State | Target User in Roster | Overlay | Iframe |
|---------------|----------------------|---------|--------|
| Connected | Present | Hidden | Solo stream playing |
| Connected | Absent (disconnected) | Visible: "{Username}" + "Disconnected" | Iframe remains (VDO.Ninja may show black/frozen) |
| Connected | Returns after disconnect | Hidden (auto-recover) | Stream resumes |
| Deactivated | N/A | N/A | Window.close() called by main companion |

### Roster Pill Visual States

| Popout Window State | Pill Appearance |
|--------------------|-----------------|
| No popout open | Default pill style (no left border) |
| Popout open and active | 2px left border --accent-connect (#40E070) |
| Popout was open but user closed it | Default pill style (window.closed check on next roster update cleans up Map) |

### OSC Video Active Feedback

| Plugin Video State | OSC Feedback Value | TouchOSC Button |
|-------------------|--------------------|-----------------|
| Inactive (no companion) | `/JamWide/video/active` 0.0 | Off (dark) |
| Active (companion running) | `/JamWide/video/active` 1.0 | On (green) |

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Roster pill click target | Pills are at least 44px wide (roster pill min-width from Phase 12). Height inherits from roster strip. |
| Roster pill keyboard access | Pills are `<button>` elements with `title` attribute. Keyboard focusable via Tab, activated via Enter/Space. |
| Popout iframe title | `<iframe title="{Username} - Video">` for screen readers |
| Popout disconnect overlay | Uses semantic heading for username (`<h2>`) and paragraph for status text |
| Popup blocked banner | Uses `role="alert"` for screen reader announcement |
| Color-only indicator | Active-popout green border is supplemented by the fact that the popout window itself is visually present on screen |
